## 一、法律自救 App (LegalShield)

### 1.1 市場規模

| 指標 | 數據 | 來源 |
|------|------|------|
| 全球 LegalTech 市場 (2025) | **354 億美元** | Future Market Insights |
| 全球 LegalTech 市場 (2035) | **725 億美元** (CAGR 7.6%) | Future Market Insights |
| 日本 LegalTech CAGR | **8.6%** (高於德國 7.8%) | Future Market Insights |
| 日本民事新受案件 (2023) | **約 140 萬件/年** | 裁判所データブック 2024 |
| 判決結案數 (2023) | **67,986 件** | 裁判所統計 |
| 「泣き寝入り」市場 (放棄維權) | **年間 6,000 億日圓** | Legal AI 株式會社推計 |

### 1.2 本人訴訟 (Pro Se) 的現狀

| 指標 | 數據 |
|------|------|
| 民事訴訟中「本人のみ」比例 | **約 7%** (裁判所データブック 2024) |
| 律師費起跳 | **10-30 萬日圓** (諮詢 + 著手金) |
| 法テラス (法律扶助) 每年利用人數 | ~30 萬件 (但僅限低收入) |

**核心痛點：**
- 年間約 **6,000 億日圓** 的糾紛因為「請不起律師」而被放棄
- 複雜案件（跨國、技術性）律師不敢接、不願接
- 偏鄉地區律師密度極低（高知幡多地區幾乎無常駐專業律師）
- 語言障礙：外國居民更難取得法律資源

### 1.3 受眾分析

| 客群 | 規模 | 痛點 | 付費意願 |
|------|------|------|---------|
| **本人訴訟者** | ~10 萬人/年 | 不會寫訴狀、找不到律師 | 高 (替代律師費) |
| **外國居住者** | 約 340 萬人 | 語言障礙 + 法律制度不熟 | 高 |
| **性暴力/DV 被害者** | 通報數 ~8 萬件/年 | 二度傷害恐懼、證據保全困難 | 中 (需公費支援) |
| **消費者糾紛** | ~90 萬件/年 (消費生活センター) | 金額小不值得請律師 | 中 |
| **勞資爭議** | ~35 萬件/年 | 怕被報復、不知道權利 | 高 |
| **不動產/買賣詐欺** | 數十萬件/年 | 證據散亂、流程複雜 | 高 |

### 1.4 競爭者分析

#### 日本既有服務

| 產品 | 類型 | 優勢 | 弱點 | 你的差異化 |
|------|------|------|------|----------|
| **Legal AI (本人訴訟 AI)** | Web 服務 (免費) | 23+ 個專用 prompt、訴狀生成 | 僅 ChatGPT wrapper、無證據保全、無離線功能 | SHA256 存證 + Multi-Agent 模擬 + 律師媒合 |
| **本人訴訟 GPT** | ChatGPT Plugin | 京大背書、e-sports 誹謗中傷特化 | 僅限 GPT 生態圈、無本地處理、隱私疑慮 | SLM 本地處理敏感資料 + 全案型涵蓋 |
| **法テラス** | 政府法律扶助 | 免費律師 (低收入限定) | 資格門檻高、等待時間長、偏鄉窗口少 | 即時可用、無收入門檻 |
| **弁護士ドットコム** | 律師媒合平台 | 律師數量最多 | 只是搜尋引擎、不提供AI輔助、要自己判斷 | AI 先整理案情再媒合 → 律師接案意願高 |
| **ココナラ法律相談** | 線上諮詢 | 便宜、門檻低 | 品質不穩、無證據管理 | 結構化存證 + 策略模擬 |

#### 全球對標

| 產品 | 市場 | 模式 | 你的差異化 |
|------|------|------|----------|
| **DoNotPay** (美) | 美國 | AI 律師, $36/年 | 日本法律專用 + 離線 SLM |
| **Harvey AI** (美) | 美國 | 律師事務所用 (B2B) | 你做 B2C (個人直接使用) |
| **Clio** (加) | 全球 | 律所管理工具 | 你面向無律師的個人 |

### 1.5 差異化定位

```
現有產品:  ChatGPT wrapper → 丟問題 → 拿建議 → 自己去處理
你的產品:  引導式問卷 → 自動存證 → 策略模擬 → 文書生成 → 律師轉介
                ↑              ↑            ↑           ↑           ↑
            降低門檻      法律效力     預判勝率     省時間       閉環服務
```

**一句話差異：「不只是 AI 聊天，是一條龍的訴訟輔助管線。」**

## 二、婦幼醫療 App (PocketMidwife)

### 2.1 市場背景

| 指標 | 數據 | 來源 |
|------|------|------|
| 日本年出生數 (2024) | **686,173 人** (史上最低) | 厚生勞動省 |
| 15 歲以下人口 | 約 **1,400 萬人** (持續減少) | 厚生勞動省 |
| #8000 (小兒救急電話) 年通話數 | **約 100 萬件** | 日本小兒科醫會 |
| 小兒科醫師數 | ~16,000 人 | 厚生勞動省 |
| 過疎地域市町村數 | **885** (全市町村的 51.5%) | 總務省 |

### 2.2 偏鄉醫療的結構性困境

```
都市 (東京 23 區):  小兒科 1,200+ 間   →  半夜急診有選擇
地方都市 (高知市):  小兒科 ~40 間       →  夜間急診 1-2 間
偏鄉 (幡多地區):   小兒科 ~3 間        →  夜間急診 0 間 (要開車 1 小時)
```

**核心痛點：**
- 偏鄉父母半夜面對孩子發燒 → **不知該等到早上還是立刻開車 1 小時去急診**
- #8000 電話 → 假日/夜間經常佔線打不通
- 現有 App 功能粗糙、資料過時、AI 零介入
- 產婦產後出血在偏鄉 → 黃金時間內能否判斷決定生死

### 2.3 受眾分析

| 客群 | 規模 | 痛點 | 付費意願 |
|------|------|------|---------|
| **偏鄉 0-6 歲父母** | ~50 萬家庭 (過疎地域) | 夜間無小兒急診 | 高 (生命安全) |
| **全國 0-6 歲父母** | ~400 萬家庭 | #8000 佔線、焦慮 | 中 |
| **孕產婦** | ~70 萬人/年 | 產後異常判斷 | 高 |
| **助產院/診所** | ~600 間 | 缺乏 AI 輔助工具 | 中 (B2B) |
| **自治體 (市町村)** | 885 過疎市町村 | 母子保健數位化需求 | 高 (B2G 標案) |

### 2.4 競爭者分析

#### 日本既有服務

| 產品 | 類型 | 優勢 | 弱點 | 你的差異化 |
|------|------|------|------|----------|
| **こどもの救急 (ONLINE-QQ)** | 網站 (厚勞省) | 日本小兒科學會監修 | 2006 年上線、純網頁選單式、無 AI、無影像辨識 | AI 對話 + Edge Vision + 離線可用 |
| **小兒救急支援 App (大阪)** | App (消防局) | 官方、免費、醫療機關地圖 | 僅大阪府、App Store 評價 2.1 星、電話號碼錯誤回報 | 全國對應 + AI Triage + 品質保證 |
| **#8000 電話** | 電話 (政府) | 有護理師接線 | 佔線嚴重、僅限電話、無記錄功能 | 24h 可用、不佔線、完整紀錄 |
| **ルナルナ / トツキトオカ** | App (民間) | 孕期記錄、社群活躍 | 無急救功能、無 Triage | 急救判斷才是核心 |
| **Kids Doctor** | 線上問診 | 真人醫師回答 | 需預約等待、收費高 | AI 即時 + 超限才轉人工 |

#### 學術基礎

> 大阪大學 Katayama 等人 (2021) 開發的小兒急救 App，4 年間使用 **59,375 次**，
> 中位年齡 **1 歲**，78.5% 為「疾病」類，最多選項為「發燒」(36.4%)。
> **證明日本父母對此類工具有明確需求。**
> (JMIR Pediatrics and Parenting, doi:10.2196/27581)

### 2.5 差異化定位

```
現有產品:  選單式問卷 → 「建議就醫」/「在家觀察」 → 結束
你的產品:  AI 問卷 → Edge Vision (相機) → Triage 分級 → 急診導航 → 就醫摘要
                            ↑                    ↑              ↑
                     影像辨識嬰兒狀態      紅/黃/綠分級    給醫師看的報告
                     (離線可用!)
```

**一句話差異：「不只告訴你該不該去醫院，還用 AI 幫你看孩子的狀況。」**

## 四、市場前景與機會

### 4.1 順風因素

| 因素 | 說明 |
|------|------|
| **日本政府「數位田園都市國家構想」** | 政府編列預算推動偏鄉數位化，B2G 標案機會大 |
| **2025 民事裁判 IT 化** | 法院推動線上訴訟，與法律 App 互補 |
| **AI 規制寬鬆** | 日本對 AI 的法規比歐盟寬鬆，有利於快速上市 |
| **少子化危機感** | 政府對母子保健投入加大，補助金來源多 |
| **SLM 技術成熟** | 2026 年 1-3B 模型已可在手機跑，邊緣運算可行 |

### 4.2 逆風因素

| 因素 | 對策 |
|------|------|
| **法律免責風險** | 嚴格定位為「工具」+ 強制引用法源 + 使用者署名 |
| **醫療法規制** | 非診斷聲明 + 對齊小兒科學會指引 + 審查委員會 |
| **用戶信任建立** | 開發者自身案例 (POC) + 學術合作 + 政府背書 |
| **偏鄉網路不穩** | Edge SLM 離線可用是核心賣點 |
| **少子化=市場縮小?** | 單價提高 + B2G 政府標案 + 海外擴張 (台灣/東南亞) |

### 4.3 收益預測 (保守估計)
