# LegalShield 平台功能界定書
## Version 0.1 — 最小可行平台 (MVP)

---

## 一、平台定位

**LegalShield 不是一個 App，而是一個三方平台：**

```
┌─────────────┐         ┌──────────────┐         ┌─────────────┐
│   被害者    │◀───────▶│   平台核心    │◀───────▶│  合作夥伴   │
│  (C 端)     │  求助   │ (App + API)  │  案件    │ (B 端機構)  │
│             │         │              │         │             │
│ • 即時存證  │         │ • 證據鏈     │         │ • 案件審查  │
│ • 緊急求助  │         │ • 法規 RAG   │         │ • 介入判斷  │
│ • 法律輔助  │         │ • 差分隱私   │         │ • 後續追蹤  │
└─────────────┘         └──────────────┘         └─────────────┘
        │
        ▼
┌─────────────┐
│   法律知識   │
│  (LLM 引擎)  │
│             │
│ • 判例檢索   │
│ • 文書生成   │
│ • 策略建議   │
└─────────────┘
```

## 二、MVP 功能清單（分層）

### Layer 1: 證據保全核心

> **核心價值：讓手機成為「司法級證據採集器」——所有原始資料不離開裝置**

| # | 部署 | 功能 | 說明 | 優先級 |
|---|------|------|------|--------|
| 1.1 | **[PHONE]** | **相機採證** | 拍照 → 即時 SHA-256 → AES-256 加密 → GPS + 時間戳水印。所有運算在 iPhone 本地完成，原始照片不上傳 | 🟢 P0 |
| 1.2 | **[PHONE]** | **錄音採證** | 同上。麥克風 → AVAudioRecorder → SHA-256 → AES-256-GCM → SwiftData | 🟢 P0 |
| 1.3 | **[PHONE]** | **證據鏈驗證** | 本地比對 hash，不需網路。驗證結果：「這張照片從未被修改」 | 🟢 P0 |
| 1.4 | **[PHONE]** | **本地加密儲存** | SwiftData (metadata) + FileSystem (加密檔案) + Keychain (金鑰)。不上 iCloud、不上後端 | 🟢 P0 |
| 1.5 | **[PHONE]** | **緊急鎖定** | 觸發後本地檔案系統設為 read-only，防止覆寫或刪除 | 🟡 P1 |

### Layer 2: 緊急安全（差異化核心）

> **核心價值：求救時，資料到達「有資格介入的人」手中**

| # | 部署 | 功能 | 說明 | 優先級 |
|---|------|------|------|--------|
| 2.1 | **[PHONE]** | **緊急轉介同意書** | 首次使用時在 iPhone 上簽署（逐條勾選）。同意書加密儲存在本地 Keychain | 🟢 P0 |
| 2.2 | **[BOTH]** | **緊急按鈕** | **PHONE**: 用戶一鍵觸發 → 本地鎖定證據 → 打包「去識別化資料包」(hash + 區域 + 類型)。**BACKEND**: 接收資料包 → 解密合作夥伴路由邏輯 → 推播至 Partner Dashboard | 🟢 P0 |
| 2.3 | **[PHONE]** | **系統警示（不自動通報）** | 求救語音關鍵詞/感測器異常 → **僅在 iPhone 本地處理** → App 內彈窗詢問用戶。不傳送任何資料到後端 | 🟡 P1 |
| 2.4 | **[BACKEND]** | **合作夥伴路由** | GPS 座標傳至後端（用戶同意後）→ 後端比對管轄區域多邊形 + 值班狀態 + 案件類型匹配 + 負載均衡 → 決定分派給哪個合作夥伴 | 🟢 P0 |
| 2.5 | **[BACKEND]** | **緊急聯繫人通知** | 後端 SMS Gateway（如 Twilio）發送簡訊。iPhone 只傳「需要發送」指令，不傳內容 | 🟡 P1 |

### Layer 3: 法律輔助（高頻使用、無個資門檻）

> **核心價值：讓一般民眾在遇到法律問題時，先有一個「不會騙你的 AI 律師助理」**

| # | 部署 | 功能 | 說明 | 優先級 |
|---|------|------|------|--------|
| 3.1 | **[BACKEND]** | **法律問答** | 用戶輸入問題 → 傳至後端 → RAG 檢索向量資料庫（LanceDB on Windows/Ollama）→ LLM 生成回答 → 回傳 iPhone。iPhone 只做 UI 渲染 | 🟢 P0 |
| 3.2 | **[BOTH]** | **契約陷阱掃描** | **PHONE**: Vision OCR 本地執行（iOS 原生，無網路延遲），提取文字。文字上傳後端（用戶同意後）。**BACKEND**: LLM 分析條款 → 標註高風險 → 回傳結果。原始圖片不上傳 | 🟢 P0 |
| 3.3 | **[BACKEND]** | **文書生成** | 後端 LLM (Ollama) 生成草稿 → 回傳 iPhone。生成過程在 RTX 4080 上完成，iPhone 無法承擔 | 🟡 P1 |
| 3.4 | **[BACKEND]** | **判例/法條檢索** | RAG 向量搜尋在後端 LanceDB 執行。iPhone 只傳 query string，不回傳整個資料庫 | 🟡 P1 |
| 3.5 | **[PHONE]** | **行政救濟期限提醒** | 本地推播（UNUserNotificationCenter）。期限資料儲存在 SwiftData，不需網路即可提醒 | 🟡 P1 |

### Layer 4: 防誘導取證（垂直領域差異化）

> **核心價值：保護弱勢者的「證詞證據能力」**

| # | 部署 | 功能 | 說明 | 優先級 |
|---|------|------|------|--------|
| 4.1 | **[PHONE]** | **即時語音轉文字** | Speech 框架本地執行（Apple 原生）。音訊不上傳，逐字稿本地生成 | 🟡 P1 |
| 4.2 | **[PHONE]** | **誘導問句偵測** | 本地 regex + CoreML 意圖分類（端側模型）。偵測結果即時顯示，零延遲、零上傳 | 🟡 P1 |
| 4.3 | **[PHONE]** | **訪談品質報告** | 本地統計計算（誘導次數、風險分數）。報告 PDF 本地生成，用戶自行選擇是否分享 | 🔵 P2 |

### Layer 5: 感測器整合（硬體生態）

> **核心價值：透過 BLE 感測器補充「行為證據」**

| # | 部署 | 功能 | 說明 | 優先級 |
|---|------|------|------|--------|
| 5.1 | **[PHONE]** | **BLE 裝置掃描** | CoreBluetooth 本地掃描。心率、環境數據本地接收，不經後端 | 🔵 P2 |
| 5.2 | **[PHONE]** | **感測器異常偵測** | 本地規則引擎（閾值比較）。異常標記本地儲存，不上傳 | 🔵 P2 |
| 5.3 | **[PHONE]** | **Mock 感測器** | Xcode Simulator 專用。本地隨機數生成，無需硬體 | 🟢 P0 |

### Layer 6: 平台管理（B 端功能）

> **核心價值：讓合作夥伴（児童相談所/DV 中心）願意使用**

| # | 部署 | 功能 | 說明 | 優先級 |
|---|------|------|------|--------|
| 6.1 | **[BACKEND]** | **Partner Dashboard** | Web App（React/Vue）部署在 AWS EC2 或 Windows Server。專員用瀏覽器登入，不在 iPhone 上 | 🟢 P0 |
| 6.2 | **[BACKEND]** | **案件路由管理** | 後端資料庫儲存合作夥伴管轄區域多邊形、值班表、負載上限。路由演算法在後端執行 | 🟡 P1 |
| 6.3 | **[BACKEND]** | **去識別化案件卡** | 後端在收到完整資料後，主動過濾：移除精確 GPS（只留區域）、移除姓名、只留 hash。Dashboard 看到的永遠是後端處理後的版本 | 🟢 P0 |
| 6.4 | **[BACKEND]** | **授權解密機制** | 專員點擊「確認介入」→ 後端記錄審計日誌 → 用獨立金鑰解密並顯示完整個資。解密在後端完成，iPhone 不儲存專員權限 | 🟢 P0 |
| 6.5 | **[BACKEND]** | **審計日誌** | 不可篡改資料庫（或區塊鏈/只追加日誌）儲存在後端。誰何時看了什麼，永久記錄 | 🟡 P1 |

### Layer 7: 研究閉環（長期護城河）

> **核心價值：用匿名數據持續改善產品，同時申請補助**

| # | 部署 | 功能 | 說明 | 優先級 |
|---|------|------|------|--------|
| 7.1 | **[PHONE]** | **差分隱私統計** | iPhone 本地計算：案件類型分布、誘導問句模式。計算時加入 Laplace noise (ε=1.0)。原始數據不出裝置 | 🔵 P2 |
| 7.2 | **[BOTH]** | **週批次上傳** | **PHONE**: 每週自動生成 DP 統計包（本地）。**BACKEND**: 經 IRB API endpoint 上傳。上傳的是噪聲統計，無法反推個人 | 🔵 P2 |
| 7.3 | **[BACKEND]** | **研究 Dashboard** | 視覺化匿名統計，供研究團隊/補助機構查看。部署在後端，iPhone 不參與 | 🔵 P2 |

## 三、MVP 範圍（Phase 1 必做 vs 不做）

### ✅ Phase 1 必做（0-3 個月）

**📱 iPhone App（本地端）**
- 案件建立與管理（SwiftData）
- 相機/錄音採證 → 即時 SHA-256 → AES-256 加密（CryptoKit + FileSystem）
- 法律問答 UI（輸入問題 → 傳至後端 → 顯示回答）
- 契約陷阱掃描：Vision OCR 本地執行 → 提取文字 → 上傳文字 → 顯示標註結果
- 緊急轉介按鈕 + 同意書本地簽署（Keychain 儲存）
- 系統警示：本地 Speech/感測器偵測 → 僅 App 內彈窗，**零上傳**
- 行政救濟期限本地推播（UNUserNotificationCenter）
- 差分隱私統計本地計算（不上傳）

**🖥️ 後端系統（AWS EC2 / Windows Server + Ollama）**
- **FastAPI 服務**：接收 App API 請求（法律問答、契約分析、緊急轉介）
- **LanceDB 向量資料庫**：日本法規 + 判例 RAG
- **Ollama LLM**：回答生成、條款分析、文書草稿（RTX 4080）
- **案件路由引擎**：管轄區域多邊形比對 + 負載均衡
- **Partner Dashboard**：Web App（React/Vue），專員瀏覽器登入
- **去識別化處理**：收到資料後主動過濾精確 GPS/姓名，只留區域 + hash
- **審計日誌資料庫**：PostgreSQL 只追加日誌表
- **SMS Gateway**：Twilio 發送緊急聯繫人通知

**🔗 兩端互動流程（範例：緊急轉介）**
```
用戶按下緊急按鈕
    ↓
📱 iPhone: 本地鎖定證據 → 打包「去識別化資料包」
    （包含：證據 hash 陣列、區域名稱、案件類型、時間戳、同意書快照）
    ↓
🖥️ 後端: 接收資料包 → 解密路由邏輯 → 比對管轄區域 + 值班表
    → 推送至 Partner Dashboard（去識別化卡片）
    → SMS 通知緊急聯繫人（如有設定）
    ↓
👤 專員: 在 Dashboard 看到案件 → 點擊「確認介入」
    → 後端記錄審計日誌 → 用獨立金鑰解密完整個資
    → 專員聯繫用戶/啟動救援
```

### ❌ Phase 1 不做（3 個月後再評估）

- ❌ BLE 感測器硬體整合（用 Mock 數據代替）
- ❌ 防誘導取證的完整 InterviewCopilot（先做簡易版）
- ❌ 裁判員制度對應指南
- ❌ 自動上傳研究數據（IRB 審查通過前不上傳）
- ❌ 付費訂閱系統
- ❌ 多語言完整支援（先做日文+繁體中文）

## 四、功能驗收標準

| 功能 | 驗收標準 |
|------|----------|
| 緊急轉介 | 用戶按下按鈕 → 10 秒內證據鎖定 → 30 秒內案件出現在 Partner Dashboard |
| 法律問答 | 用戶問「児童虐待 通報 期限」→ 5 秒內回應，引用児童虐待防止法第3条 |
| 契約掃描 | 拍攝「放棄一切權利」條款 → 3 秒內標紅，提示「消費者契約法第10条違反可能性」 |
| Partner Dashboard | 專員登入 → 看到當日案件 → 點擊「確認介入」→ 看到完整資料 + GPS |

## 五、與 Toyota 財団補助申請的對應

補助申請書中強調以下功能：

1. **「児童虐待通報のデジタル化」** → 緊急轉介 + Partner Dashboard
2. **「現場業務の負担軽減」** → 自動案件路由 + 去識別化案件卡
3. **「モビリティ×安全」** → GPS 定位 + BLE 求救按鈕（未來）
4. **「データ駆動型改善」** → 差分隱私統計（匿名化後共享）

> **申請重點：不是「我們做了一個很酷的 App」，而是「我們讓児童相談所少打 50% 的電話、少填 30% 的紙本」**
