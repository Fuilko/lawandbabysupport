# LegalShield 研究驅動開發路線圖
## 法規影響權重矩陣 × 日本優先策略 × 研究-產品-補助三位一體

---

## 一、核心命題

> **不是「法律領域越多越好」，而是「哪個法規缺口 × 技術可解性 × 社會影響力」的交集最大。**

我們的優勢不是知道所有法律，而是「**在事件發生的第一時間，用感測器 + 存證 + AI 鎖定不可逆的證據**」。

所以開發規劃必須回答三個問題：
1. **哪些法規領域的受害者最需要「即時存證」？**
2. **哪些領域的研究補助最容易拿到？**
3. **真實數據如何能回流到產品，形成護城河？**

## 二、補助戰略：企業財團 CSR vs 政府學術補助

> **核心判斷：走丰田財団（企業 CSR）為主軸，政府學術補助為副軸。**

| 維度 | 企業 CSR / 財團（如トヨタ財団） | 政府學術補助（如 KAKENHI） |
|------|----------------------------------|---------------------------|
| **審查速度** | 3-4 個月 | 6-12 個月（甚至 18 個月） |
| **申請門檻** | 中等，重視社會影響力 | 高，需既有學術成果、IRB、嚴謹方法論 |
| **自由度** | 高，可包含產品開發、硬體原型 | 低，限學術研究，產品化需另尋資金 |
| **持續性** | 可多期連續申請（トヨタ財団慣例） | 通常一次性，結案後需重新申請 |
| **金額規模** | 中等（300-800 萬日圓/期） | 大額（基盤研究可達 5,000 萬日圓） |
| **對產品幫助** | 可直接資助 App 開發、感測器硬體 | 僅資助論文發表、研討會 |
| **品牌背書** | 強（トヨタ→安全・信頼） | 學術權威，但一般民眾認知較弱 |

**為什麼トヨタ財団特別適合 LegalShield：**

1. **戰略親和性**：トヨタ的核心價值是「安全」（Safety）與「信頼」（Trust），與 LegalShield 的「被害者支援」「證據保全」完全共振
2. **技術親和性**：トヨタ財団近年積極資助「モビリティ×安全×社会課題」的交叉研究，BLE 感測器、即時數據分析正好符合
3. **速度優勢**：學術補助從申請到撥款可能要 18 個月，トヨタ財団通常 3-4 個月即可決定——對 startup 的現金流至關重要
4. **硬體開發可納入**：トヨタ CSR 基金允許「原型製作費」，這意味著 IR Dongle 的硬體開發也能獲得資助
5. **日本市場背書**：「トヨタ財団助成事業」的標籤在日本社會具有極高公信力，遠勝一般 startup

**戰術建議**：
- Phase 1 先申請トヨタ財団「国内助成」（一般 300-500 万円）建立信譽
- 取得トヨタ財団背書後，Phase 2 同時申請 KAKENHI（基盤 C / 若手研究）
- 雙軌並行：企業 CSR 養產品，政府補助養論文

## 三、法規影響權重矩陣 (Legal Impact Weight Matrix)

### 評估維度 (四軸)

| 維度 | 權重 | 說明 |
|------|------|------|
| **受害者脆弱度 (Vulnerability)** | 30% | 兒童、老人、身心障礙者、外勞、契約弱勢方 |
| **取證困難度 (Evidence Difficulty)** | 25% | 隱蔽性高、證人難尋、機構內部封閉 |
| **技術可解性 (Tech Solvability)** | 25% | BLE 感測器、相機、麥克風、GPS 能否介入 |
| **市場/補助規模 (Market Scale)** | 20% | 潛在用戶數、政府補助額度、NGO 合作機會 |

### 日本法規領域評分

| 領域 | 脆弱度 | 取證難 | 技術解 | 市場規模 | **總分** | **優先級** |
|------|--------|--------|--------|----------|----------|------------|
| 兒少虐待 (児童虐待防止法) | 10 | 10 | 9 | 8 | **9.15** | 🥇 Phase 1 |
| 偷拍/跟蹤 (ストーカー規制法) | 9 | 9 | 10 | 7 | **8.75** | 🥇 Phase 1 |
| 勞動權益 (労働基準法) | 8 | 8 | 7 | 9 | **7.90** | 🥈 Phase 2 |
| 消費者陷阱 (消費者基本法) | 7 | 7 | 8 | 10 | **7.90** | 🥈 Phase 2 |
| 高齡者虐待 (高齢者虐待防止法) | 10 | 8 | 6 | 6 | **7.65** | 🥉 Phase 3 |
| 行政救濟 (行政手続法/訴訟法) | 6 | 9 | 5 | 7 | **6.65** | 🥉 Phase 3 |
| 無人機違法 (航空法) | 5 | 7 | 9 | 4 | **6.10** | Phase 4 |
| 環境犯罪 | 6 | 6 | 8 | 3 | **5.65** | Phase 4 |

### 為什麼「日本優先」

| 因素 | 日本 | 台灣 |
|------|------|------|
| 補助額度 | 内閣府・文科省・法務省 多管道大額 | 相對分散，數位部額度有限 |
| 研究基礎設施 | 国立研究開発法人、大学共同利用機関 | 需自行建置 |
| 數位取證法規 | 電磁的記録に関する法律 (平成16年) 已成熟 | 電子簽章法仍在適應期 |
| 隱私保護 | 個人情報保護法嚴格 → 產品設計必須極致安全 → 反而成為競爭優勢 | 相對寬鬆 |
| NGO 協作網絡 | NPO法人枠組み成熟，可快速試驗 | 仍在發展 |
| **關鍵：日本法院對「即時存證」採信度更高** | ✅ | ⚠️ |

## 四、現有架構缺口分析

### 已具備 ✅

- [x] BLE 感測器抽象層
- [x] 防誘導訪談
- [x] LLM API 橋接

- [x] 緊急模式

### 缺口 🚨

| 缺口 | 嚴重度 | 阻塞 Phase | 解決方案 |
|------|--------|-----------|----------|
| **日本法規知識庫 (RAG)** | 🔴 高 | 所有領域 | 建 vector DB：児童虐待防止法、ストーカー規制法、労基法、消費者基本法 |
| **行政救濟流程管理** | 🟠 中 | 行政法 | 新模組：行政手続期限提醒、訴願/訴訟時程 |
| **研究倫理 (IRB) 審查** | 🔴 高 | 數據收集 | 任何「真實案件數據」回傳前，需通過研究倫理委員會 |
| **補助款追蹤模組** | 🟡 低 | 產品化 | 非核心，但對補助申請有利 |
| **多語言法規對照** | 🟠 中 | 跨境 | 日/台法規條文對照（如児童虐待防止法 vs 兒少保護法） |
| **匿名化統計上傳** | 🟠 中 | 研究閉環 | 需設計差分隱私 (Differential Privacy) 機制 |
| **合議制/陪審員制度對應** | 🟡 低 | 訴訟法 | 日本裁判員制度下的證據呈現格式 |

## 五、研究-產品-補助三位一體架構

```
                    ┌─────────────────┐
                    │   研究機構/大學  │
                    │  (法務研究科)    │
                    └────────┬────────┘
                             │ 提供法規知識、研究倫理審查
                             ▼
┌──────────┐    回饋      ┌─────────────────┐    補助申請    ┌──────────┐
│  真實案件 │◀── 匿名統計 ──│   LegalShield   │──────────────▶│ 政府補助 │
│  數據庫  │               │    iOS App      │   (技術+社會)  │  機構    │
└──────────┘               └─────────────────┘               └──────────┘
                             │ 產品輸出
                             ▼
                    ┌─────────────────┐
                    │   弱勢使用者    │
                    │ (被害者/家長/   │
                    │  勞工/消費者)   │
                    └─────────────────┘
```

### 閉環機制 (Research Loop)

```
Step 1: 產品使用
  → 用戶在 App 內完成取證、訪談、掃描

Step 2: 去識別化數據提取 (Local)
  → App 端將案件數據轉為結構化統計（移除姓名、地點、內容）
  → 僅保留：案件類型、證據數量、誘導問句次數、AI 評分、結果

Step 3: 差分隱私處理 (Differential Privacy)
  → 加入 Laplace noise ε=1.0
  → 確保無法從統計反推個別用戶

Step 4: 上傳研究伺服器
  → 經 IRB 審查通過的 API endpoint
  → 每週批次上傳，非即時

Step 5: 研究分析
  → 研究團隊分析「哪類誘導問句最常出現」
  → 「哪類機構的異常感測器模式最明顯」
  → 「哪類契約條款的無效主張成功率最高」

Step 6: 模型更新
  → 研究發現 → 更新 LLM prompt / 規則引擎
  → 例如：發現「XX 類幼兒園」的虐待案件中，85% 在首次訪談時出現「身體部位暗示」
  → → 強化 InterviewCopilot 對此類問句的偵測權重

Step 7: 產品迭代
  → 新版本推送給所有用戶
  → 更好的偵測 = 更好的證據 = 更高的勝率 = 更多用戶

Step 8: 補助申請
  → 用真實數據證明社會影響力
  → 申請下一期研究補助
  → 資金支持更深入的學術研究
```

## 六、日本 Phase 開發規劃

### Phase 1 (0-6 月)：兒少虐待 + 偷拍 (核心場景)

**研究目標**：
- 與日本某大學法務研究科合作，建立「児童虐待におけるデジタルフォレンジクス」研究框架
- 收集 50 例「首次揭露」訪談數據，分析誘導問句分布

**產品目標**：
- 完成 InterviewCopilot 日文支援
- 完成 BLE 心率手環 + 求救按鈕日本市場適配
- App Store Japan 上架

**補助管道（主軸：企業財團 CSR）**：
- 🥇 **トヨタ財団 (Toyota Foundation)** — 国内助成「多様性の実現に向けた社会システム」
  - 児童安全・被害者支援に強い興味。デジタル技術×社会課題の研究を優遇
  - 申請→決定 約 3-4 ヶ月（政府補助より速い）
- 🥈 **トヨタ CSR 基金** — 「こども安全」プロジェクト助成
  - 児童虐待防止・交通安全の横断的CSR。ハードウェア開発も対象
- 副軸：内閣府「こども家庭庁」數位化推進補助（政府補助は補完的に）

### Phase 2 (6-12 月)：勞動權益 + 消費者陷阱

**研究目標**：
- 分析「過労死ライン」與感測器數據的相關性
- 消費者契約陷阱條款的無效主張成功率統計

**產品目標**：
- ContractTrapAnalyzer 上線
- 勞動時數感測器串接 (打卡記錄 OCR)
- 訴訟法模組：期限提醒、裁判員制度對應

**補助管道（主軸：企業財團 CSR）**：
- 🥇 **トヨタ財団** — 継続助成「労働とテクノロジーの交差点」
  - 過労死・働き方改革のデータ駆動型研究に適合
- 🥈 **Nippon Foundation / 日本財団** — 社会的包摂テクノロジー助成
  - 労働者・消費者の「情報弱者」支援を強化
- 副軸：厚生労働省「過労死防止対策推進補助金」

### Phase 3 (12-18 月)：高齡者虐待 + 行政救濟

**研究目標**：
- 高齡者虐待の「行動不能」感測器モード
- 行政救済における「証拠能力」のボトルネック

**產品目標**：
- 跌倒偵測感測器整合
- 行政手続期限管理模組

**補助管道（主軸：企業財團 CSR）**：
- 🥇 **トヨタ財団** — 「高齢化社会における安全と尊厳」
  - 高齢者虐待防止・介護現場のデジタル化。BLEセンサーネットワークが親和性高い
- 🥈 **いちょう財団 (Ginkgo Foundation)** — 高齢者虐待防止 NGO 連携助成
- 副軸：内閣府「高齢社会対策」研究開発助成

### Phase 4 (18-24 月)：無人機 + 環境 + 跨境

**研究目標**：
- 跨境證據鏈互通（日台法規對照）
- 環境感測器データの裁判所採信標準

**補助管道（主軸：企業財團 CSR）**：
- 🥇 **トヨタ財団** — 「環境・安全・モビリティの統合」
  - 無人機×環境モニタリング。交通安全にも応用可能
- 🥈 **航空局 / 国土交通省連携** — ドローン規制緩和実証事業
- 副軸：環境省「環境研究総合推進費」

## 七、關鍵風險與對策

| 風險 | 對策 |
|------|------|
| **IRB 審查不通過** | 預設「數據不上傳」模式，IRB 通過後才開啟研究功能 |
| **日本律師會反對** | 定位為「被害者支援工具」而非「法律服務」，不搶律師飯碗 |
| **個資法違反** | 所有原始數據端內處理，上傳僅為 ε-差分隱私統計 |
| **法院不採信** | 與法務研究科合作發表論文，建立證據能力學術背書 |
| **補助被砍** | 保持產品自給能力（App 訂閱 + 硬體銷售）|

## 八、下一步行動清單

1. **聯繫日本法務研究科** — 確認合作意願與 IRB 流程
2. **擴充 RAG 知識庫** — 加入日本核心法規（児童虐待防止法、ストーカー規制法等）
3. **設計差分隱私模組** — 任何統計上傳前必須經過 DP 處理
4. **申請第一筆補助** — 以 **トヨタ財団「国内助成」** 為目標，準備 proposal（主題：「児童虐待におけるデジタルフォレンジクスとモビリティ×安全の統合」）。3-4 個月決定，快速建立信譽
5. **建立研究數據 dashboard** — 讓研究團隊能視覺化查看匿名統計
6. **發表第一篇論文** — 目標：Digital Investigation 或 Computer Law & Security Review
