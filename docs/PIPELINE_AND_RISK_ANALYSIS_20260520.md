## Part 2 — 新增功能：盗撮 / 跟蹤騷擾 / 加害者追蹤揭露

### 2.1 三個功能分開看（非常重要）

很多人把這三個混為一談，但**法律風險完全不同等級**：

| 功能 | 角色 | 法律性質 | 開發風險 |
|------|------|----------|----------|
| **A. 反盗撮（防衛性）** | 受害者掃描自己空間 | **無爭議** | 🟢 低 |
| **B. 跟蹤騷擾證據蒐集（記錄型）** | 受害者記錄發生在自己身上的事 | 自我證據保全 | 🟡 中（但可控） |
| **C. 加害者追蹤揭露（攻擊性）** | 對加害者進行反向蒐證 / 公開 | **高度違法地雷** | 🔴 極高 |

**戰略建議：**「A + B 全力做，C 永遠不做。」C 即使做也只能做「協助交給警察 / 媒體 / 律師」的中間層，**絕對不能做公開揭露**。

### 2.2 功能 A：反盗撮（已存在 AntiSurveillanceView）

**現況：** `Views/AntiSurveillanceView.swift` 已有 Wi-Fi / 磁力計 / LiDAR / 紅外線四模式。

**可加強：**

| 項目 | 內容 |
|------|------|
| Wi-Fi 異常 IoT 偵測 | 比對 OUI 廠商表，標記「IP camera / spy device」常見 MAC prefix |
| BLE Beacon 掃描 | AirTag / Tile 反追蹤（iOS 已內建警告，可做被動掃描補強） |
| 磁力計熱點圖 | 在房間內走一圈產生熱力圖，異常磁場 → 可疑點位 |
| 異常 USB / Lightning | 提示用戶檢查充電線是否被換成有 GPS 的山寨品 |
| 鏡面反射偵測（相機 + flash） | 用閃光燈 + AI 找鏡頭反光點（已在開源圈成熟） |

**法律：** 完全合法。掃描自己的空間 = 自衛。

### 2.3 功能 B：跟蹤騷擾證據蒐集

**已存在元件：** `EvidenceManager`, `AuditLog`, `SecondaryVictimizationProtection`

**新增建議：**

| 模組 | 功能 | 法律前提 |
|------|------|----------|
| **遭遇日誌（Incident Journal）** | 時間/地點/距離/截圖/錄音/錄影 一鍵打包 | 受害者本人記錄 = 合法 |
| **GPS 軌跡反向比對** | 受害者自身 GPS log（**只記錄自己**）→ 與多次「巧遇」地點比對，量化跟蹤可能性 | 只 track 自己，無爭議 |
| **通訊證據匯入** | LINE / SMS / Email / SNS DM 截圖 + OCR + Hash | 已有 `CommunicationEvidenceImporter.swift` |
| **公開騷擾辨識** | SNS 上對自己的留言 → 抓取 + 鎖定（不主動爬加害者） | 公開內容 + 自己被指名 = 合法 |
| **報案文件自動生成** | 依 ストーカー規制法 第 2 条要件，自動填申告書草稿 | 只是模板，無爭議 |
| **接觸警告（Warning Letter）** | 第一段：律師受任の上、内容証明発送パッケージ | 需配合律師（避免非弁） |

**法律重點：**

```
ストーカー規制法（平成12年法律81号）

第2条：「つきまとい等」八類型
  1. つきまとい・待ち伏せ・押し掛け・うろつき
  2. 監視している旨の告知
  3. 面会・交際の要求
  4. 著しく粗野・乱暴な言動
  5. 無言電話・連続電話・FAX・メール・SNS
  6. 汚物等の送付
  7. 名誉を害する事項の告知
  8. 性的羞恥心を害する事項の告知

第3条：「ストーカー行為」= 同一の者に対し、つきまとい等を反復して行うこと
第4条：警告（公安委員会）
第5条：禁止命令
第18条：罰則（1年以下の懲役 or 100万円以下の罰金、禁止命令違反は2年）

App 的角色 → 幫受害者把「反復」具象化（時間軸 + 地圖 + 證據鏈）
                           ↓
              交給警察 → 公安委員会 → 警告/禁止命令
```

### 2.4 功能 C：加害者追蹤揭露 ⚠️ 重點警告

**先給結論：不要做。** 真的要做，請走「協助型」而非「執行型」。

#### 2.4.1 典型「想做」的功能 vs 法律對應

| 想法 | 法律問題 | 可能罪名 |
|------|----------|----------|
| 「自動人臉辨識比對加害者」 | 個資法 + 顔識別ガイドライン | 個情法違反、肖像権侵害 |
| 「對加害者反向 GPS 追蹤」 | ストーカー規制法（你變成加害者） | ストーカー罪 |
| 「在 SNS 公開加害者照片+名字」 | 名誉毀損 + 侮辱罪 + プライバシー | **刑法 230 條 3 年以下懲役** |
| 「Discord/LINE 抓加害者帳號活動」 | 不正アクセス禁止法 | **3 年以下懲役** |
| 「眾包通報，眾人標記」 | 集団的名誉毀損 | 刑事 + 高額民事 |
| 「給加害者打分數、警告其他人」 | 営業誹謗 / 信用毀損 | 刑法 233 條 |
| 「自動 OSINT（找加害者公司、家人）」 | プライバシー権 + 個情法 | 民事數百萬 + 公訴 |
| 「不經律師代寫 ‘示談要求書’ 寄給加害者」 | **弁護士法 72 條** | **2 年以下懲役 or 300 萬罰金** |
| 「替受害者向加害者談賠償」 | 同上 | **同上** |

#### 2.4.2 真實判例參考

```
【東京地判 平成27年（2015）9月8日】
SNS 上「○○は痴漢です」投稿 → 真実でも公然性あり → 名誉毀損成立
（刑法230条の2 真実性の証明 ＝ 公益目的＋真実 だが「個人の私的紛争」は
 公益性否定されることが多い）

【最判 平成6年（1994）2月8日 ノンフィクション「逆転」事件】
真実を書いても、被報道者のプライバシー権 vs 表現の自由の比較衡量で
プライバシー優先 → 損害賠償命令

【東京地判 令和2年（2020）某月】
「探偵業の届出」なしに加害者調査 → 探偵業法違反（罰金）

【最判 令和4年（2022）某月 SNS 同定行為】
個人特定に至る投稿の連鎖 → 個情法 第19条不適正利用に該当する解釈例
```

#### 2.4.3 「協助型」中間層的安全設計

如果一定要碰 C，**唯一安全的形態**：

```
                    ❌ 不能做的：App 公開加害者
                    ✅ 可以做的：把證據打包交給有資格的人

   ┌─────────────────────────────────────────┐
   │ 受害者  ─→  App 整理證據包                │
   │             ├─ 影像/錄音 hash + timestamp │
   │             ├─ 時序圖                    │
   │             └─ 法令適用建議書（AI 草稿）   │
   └────────────┬────────────────────────────┘
                │
                ▼  受害者「主動授權」（個資法第三者提供同意）
                │
   ┌────────────┴────────────────────────────┐
   │ 警察  /  弁護士  /  支援団体（NPO）        │
   │ ↑ 由他們決定後續（開示請求、刑事告訴…）     │
   │ ↑ App 不參與「揭露」決定                  │
   └─────────────────────────────────────────┘

關鍵設計原則：
（i） App 內 NEVER 顯示加害者個資給「受害者以外的人」
（ii）「公開」按鈕直接拿掉，只留「匯出給警察/律師」
（iii）任何「比對加害者身份」必須有警察/律師 token 才能解鎖
（iv）記錄「誰在何時授權誰看了什麼」（這也是現有 AuditLog 的延伸）
```

## Part 3 — 開發者本身的行政阻礙與被告風險

### 3.1 風險地圖

```
                        ┌──────────────┐
                        │ 你（開發者）  │
                        └──────┬───────┘
                               │
       ┌───────────────────────┼─────────────────────────┐
       │                       │                         │
  ┌────▼────┐           ┌──────▼──────┐          ┌──────▼──────┐
  │ 行政方   │           │ 加害者方     │          │ 第三方       │
  ├─────────┤           ├─────────────┤          ├─────────────┤
  │ 個情委  │           │ 名誉毀損告訴 │          │ 誤判被害者   │
  │ 警察庁  │           │ プライバシー │          │ 同名誤認     │
  │ 弁護士会│           │ 削除請求     │          │ App Store    │
  │ 公安委  │           │ 仮処分       │          │ Apple/Google │
  │ 探偵協会│           │ 民事数百万   │          │ 媒體誤報     │
  │ 国税    │           │              │          │              │
  └─────────┘           └─────────────┘          └─────────────┘
```

### 3.2 行政方阻礙（依機關別）

| 機關 | 可能介入理由 | 對策 |
|------|------------|------|
| **個人情報保護委員会** | App 處理大量個資 → 安全管理措置 / 委託先監督 / 越境移転 規制 | 法27条同意、24条越境同意（伺服器在日本就免）、規程整備 |
| **警察庁 / 都道府県警** | 「素人が捜査の真似事」「刑事手続を妨害」 | App 永遠**只記錄、不偵查**；明示「報案請洽 110」 |
| **法務省 / 弁護士会** | **弁護士法 72 條「非弁活動」** | App 不出具「法律意見」；只提供「資訊整理」+「請洽律師」CTA |
| **公安委員会** | 「私的監視を煽動」 | 對 C 類功能（追蹤加害者）絕對不做 |
| **探偵業協会 / 警察生活安全課** | **探偵業法 第 4 条 屆出義務違反 = 罰金** | 不做有償「身份調査」業務；如要做，先做屆出 |
| **総務省（電気通信事業）** | 通信内容截取功能 | 不做通信代理；只處理用戶**自己手機上**的資料 |
| **消費者庁** | 不実な宣伝（「証拠能力 100%」「警察採用」等） | 文案校稿，不誇大 |
| **国税庁** | 補助金 → 課税扱い、収益活動性 | 帳簿分離（公益事業 vs 商業 SaaS） |
| **Apple App Review** | Surveillance / Tracking / Privacy 嚴審 | 文案不要出現 "track perpetrator"；定位為 "personal safety log" |

### 3.3 被告風險（依案由別）

#### 3.3.1 刑事告訴可能罪名

| 罪名 | 法條 | 法定刑 | 觸發情境 |
|------|------|--------|---------|
| 名誉毀損 | 刑法 230 | 3 年以下懲役 / 50 萬罰金 | 公開揭示加害者「事実」 |
| 侮辱 | 刑法 231 | 1 年以下懲役 / 30 萬罰金（R4 改正） | 不指明事実但侮辱 |
| 信用毀損 / 業務妨害 | 刑法 233 | 3 年以下懲役 / 50 萬罰金 | 加害者經營事業 → 影響營業 |
| 個人情報保護法違反 | 個情法 178-180 | 1 年以下 / 100 萬（法人 1 億） | 不適切な取扱、漏洩 |
| 不正アクセス禁止法 | 同法 11-12 | 3 年以下 / 100 萬罰金 | 抓加害者 SNS 帳號內部資料 |
| ストーカー規制法（反向） | 同法 18 | 1 年以下 / 100 萬罰金 | 對加害者反向「つきまとい等」 |
| 弁護士法違反（非弁） | 弁護士法 72・77 | 2 年以下 / 300 萬罰金 | 報酬を得る目的で法律事務代理 |
| 探偵業法違反 | 同法 17 | 6 月以下 / 30 萬罰金 | 屆出なしで業として調査 |
| 児童ポルノ禁止法 | 同法 7 | 1 年以下 / 100 萬（製造 5 年） | **盗撮证据若涉未成年→暫存即違法** |

#### 3.3.2 民事請求預想

```
原告（加害者 / 加害者家族 / 誤判第三者）
  ├─ 名誉毀損損害賠償（人格権侵害）            数十～数百万円
  ├─ プライバシー権侵害損害賠償                 同上
  ├─ 肖像権侵害損害賠償                         同上
  ├─ 営業權妨害損害賠償（加害者が経営者の場合） 数百万～数千万円
  ├─ 削除請求 / 投稿者情報開示請求              プロバイダ責任制限法
  └─ 仮処分（仮差止め・削除）                   早期にサービス停止リスク

【特别注意：集団訴訟リスク】
SNS で「我是受害者，這個 App 把加害者公開了」風向反轉 →
被誤指認的人或加害者家族 提集団訴訟 → サービス継続不能
```

#### 3.3.3 致命陷阱：児童ポルノ禁止法 第 7 条

```
⚠️ 開發者最大法律地雷 ⚠️

盗撮被害的證據若涉未成年（哪怕受害者是 17 歲少女自己上傳自己被偷拍的影像）：

  ↓
児童ポルノ禁止法 第 7 条第 1 項「製造」
児童ポルノ禁止法 第 7 条第 2 項「単純所持」（所持のみで処罰）
  ↓
App 後端「暫存」一秒鐘 = 持有 = 犯罪

→ 受害者上傳即觸法
→ App 開發者「提供保管場所」可能成立幇助
→ 即使受害者本人也可能被處罰（過去判例）

【唯一安全做法】
A) 影像 NEVER 離開受害者裝置（端末完結）
B) 雲端只存「Hash + Metadata」，不存原檔
C) 影像若需提交警察，由「警察的接收端」直接 SFTP / 警察提供之 system
D) App 內顯著警告：「未成年関連は警察に直接ご相談を」
E) 自動偵測「可能含未成年」→ 強制本地化 + 警告

```
Q: 你的 App 是不是在做加害者調查？
A: 否。本 App 是「受害者個人事實記錄工具」（personal incident journal）。
   功能僅限於協助受害者整理發生在自己身上的事件，並依照法律規定，
   引導其向警察、弁護士、支援団体諮詢。
   App 不對加害者進行任何主動偵查、追蹤、揭露。

Q: 你的 App 是不是在做法律諮詢？
A: 否。本 App 提供的是「公開法令資訊與判例的檢索整理」，
   不構成「特定事案に対する法律意見」。
   每個畫面都顯著標示「これは法律助言ではありません。
   弁護士にご相談ください」。

Q: 你的 App 是不是在做探偵業？

   + ISO 27001 同等の体制。
