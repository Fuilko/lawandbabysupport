## 5. FEWN（Forensic Evidence Witness Network）技術仕様

### コア理念
> *被害当事者が顔写真や位置情報を一切サーバに送らないまま、同じ加害者を記録した別の被害者と暗号学的にのみ出会える分散型ネットワーク*

### 技術スタック
| 要素 | アルゴリズム | ライブラリ |
|---|---|---|
| 加害者識別子 (電話 / 口座 / URL / 声紋) のハッシュ | **HKDF + Salt** | `cryptography` (Py) |
| Set 比較 | **Bloom Filter** または **PSI** | `pybloom-live` / `OpenMined PSI` |
| 「同じ加害者持ち」を漏らさず判定 | **PSI-Cardinality** | Google `private-join-and-compute` |

### 公証人モデル（Callisto 方式）
- CALL4 が「公証人サーバ」役
- 端末同士は直接やりとりしない
- 公証人は集合演算結果（マッチありなし）のみ知る
- マッチ時、両被害者に「他にもいる」とだけ通知（個別証拠は非開示）
- **法的助言 / 連名告訴打診 は CALL4 の弁護団経由**

### Demo シナリオ

```bash
# 被害者 A（Mapry 詐欺）
$ python fewn_demo.py register --name "victim_A" \
    --evidence "phone:+81-90-XXX-XXXX" \
    --evidence "company:株式会社Mapry" \
    --evidence "url:mapry.jp"

# 被害者 B（架空）
$ python fewn_demo.py register --name "victim_B" \
    --evidence "phone:+81-90-YYY-YYYY" \
    --evidence "company:株式会社Mapry"

# CALL4（公証人役）
$ python fewn_demo.py match
🚨 マッチ検出: victim_A ⇔ victim_B
  共通加害者: 1 件
  個別証拠は両者とも非開示（ハッシュのみ照合）
  推奨アクション: 連名告訴の打診を CALL4 経由で
```
