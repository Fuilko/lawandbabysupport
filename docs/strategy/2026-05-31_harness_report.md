## 1. 何が問題だったか（一言で）

> **「データベースは完璧なのに、AIがそれを見ずに、自信満々に“それっぽい嘘”を答えていた」**

診断の結果、原因はDBでも検索精度でもなく、**アプリの配線**でした。

| 層 | 状態 |
|---|---|
| 法令・判例データベース | ✅ 正常（中身は揃っている） |
| バックエンドの検索付き回答 `/rag/query` | ✅ 正しく動いていた（が誰も呼んでいなかった） |
| **iOSアプリのチャット** | ❌ 検索を一切せず、生のLLM（`/api/generate`）を直接叩いていた |

→ つまりAIは**記憶だけで回答**しており、条文番号も判例も金額も**創作**できる状態でした。これは法的弱者を守るアプリとして致命的です。

## 2. どう直したか（ロジックだけ — CALL4説明用）

「賢いAIを作る」のではなく、**AIが嘘をつけない“構造”を作る**というアプローチです。Harvard系リーガルテック（Casetext等）と同じ思想です。

```
ユーザーの質問
   │
   ▼
① まず質問の「種類とリスク」を判定
     例：時効・署名など“取り返しのつかない”話 → 自動的に「弁護士必須」フラグ
   │
   ▼
② 必ず先にデータベースを検索（検索ゲート）
     法令＋判例を引いてくる。引けなければ次へ進めない
   │
   ▼
③ AIには「引いてきた資料だけで答えろ。各文に出典番号[S1]を付けろ」と強制
     資料が無ければ AIは「答えられない」と言う（嘘をつくより正直を優先）
   │
   ▼
④ 自己チェック：AIの回答に出てくる条文番号・金額が
     本当に資料の中にあるか機械的に照合。無ければ「未裏付け」と赤旗
   │
   ▼
⑤ 画面に必ず表示：
     ・どの資料を根拠にしたか（出典タグ）
     ・信頼度（5段階）
     ・「弁護士に相談すべき」サイン
     ・未裏付けの主張があれば警告
   │
   ▼
⑥ 全部の質問・回答を改ざん不可能な記録（ハッシュ連鎖）で保存
```

### 核心の発想
- **ゼロ幻覚は不可能**。だから「**幻覚が起きても必ず検出・可視化される**」構造にする。
- AIに「嘘つかないで」とお願いするのは無意味。**検索しないと回答できない／出典が無いと信頼度が落ちる**という仕組みで縛る。
- 「**わからない**」と正直に言うことを許容する（むしろ評価する）。

## 3. 実際の挙動（テストで確認済み）

| 入力 | システムの反応 |
|---|---|
| 正しい根拠付きの回答 | 信頼度 **5/5**、未裏付けゼロ |
| 「民法第999条で1000万円が確定」（資料に無い創作） | **自動検出**：「未裏付け」赤旗、信頼度 0.4 に低下、弁護士フラグ点灯 |
| 検索ヒットゼロ | AIは**回答を拒否**し「専門窓口へ」と案内（信頼度 0.0） |
| 「時効前に和解契約に署名すべき？」 | **不可逆判断**と認識、弁護士相談を強制表示 |

## 4. CALL4 面談で聞きたいこと（提案）

技術の説明は最小限にして、**ロジックと運用**について意見を伺うのがよいと思います。

1. **公証人モデル（FEWN）との接続**：被害者マッチングで「同じ加害者」を暗号的に照合する仕組み（先日完成）について、CALL4が“公証人”役を担う構想は現実的か？
2. **「弁護士必須」ラインの引き方**：AIが自動で「これは弁護士へ」と判断する基準を、実務家としてどう設計すべきか？
3. **公共訴訟ブリッジ**：このシステムが生成した案件サマリーを、CALL4の公開訴訟記事フォーマットへ繋ぐ際の留意点（名誉毀損回避など）は？
4. **責任分界**：AIが「未裏付け」と表示しても利用者が行動した場合の、運用上のリスクヘッジについて。
