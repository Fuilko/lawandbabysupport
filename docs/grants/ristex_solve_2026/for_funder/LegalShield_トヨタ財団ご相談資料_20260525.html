第 0 章　お持ちする 1 ページサマリー（最初の 90 秒用）

LegalShield は、地方在住・外国籍・経済的に余裕のない方々の「自己防衛・証拠保全・法律情報アクセス」を、AI と暗号学的ネットワークで支える MIT/Apache 2.0 完全オープンソースの社会基盤プロジェクトです。
既存の司法支援制度（弁護士会・法テラス・トラブル110番等）と競合するのではなく補完する「制度の隙間を埋める市民技術（シビックテック）」として設計しています。
開発者本人がある製品紛争のご相談を 10 件以上の専門家に重ねるなかで気づいた「地方在住・外国籍当事者が直面する司法アクセスのギャップ」を、技術と公益ネットワークで埋める社会実装研究です。当該紛争は第二東京弁護士会仲裁センターにて令和 8 年（仲）第 129 号として正式受理されており、システムが現場で機能するかを検証する実証フィールドとなっています。

第 1 章　ご面談の目的

本日のご面談では、以下の 3 点についてトヨタ財団様のご見解を賜りたく存じます。

  1. 助成プログラム選定のご相談
    ─ 国内助成プログラム／研究助成プログラム／特定課題（例：「先端技術と共創する未来」）のうち、本プロジェクトに最も適合する枠についてご教示いただけないか。

  2. 申請書類の方向性についてのご意見
    ─ 採択傾向、評価軸（社会的インパクトと技術革新性の比重等）、過去の類似採択案件の有無。

  3. 共同実施体制への助言
    ─ 大学（高知大学）／NPO（フリーランス・トラブル 110 番／つくろい東京ファンド等）／弁護士団体（第二東京弁護士会仲裁センター／CALL4 等）の組合せが、トヨタ財団様の助成スキームと相性が良いか。

第 2 章　プロジェクト概要

2.1　ワンセンテンス

データ駆動型・暗号学的協調検知による「個人がアクセスできる司法」を実装する OSS 社会インフラ。

2.2　解決する社会課題

課題 現状の壁 LegalShield のアプローチ
DV／ストーカー被害者の証拠保全 警察相談前の証拠喪失・改ざんリスク 端末側 SHA-256 ハッシュチェーン + タイムスタンプ
外国籍被害者の言語・制度障壁 日本語法律用語の理解、書類起案不能 オンデバイス多言語 LLM／自動書面起案
フリーランス／代理店被害 単独立証困難、加害者の組織性が見えない 暗号学的私的集合交差（PSI）で「同じ加害者・別の被害者」と安全に出会う
地方在住者の弁護士アクセス 高知・離島等で専門家不在、出張費負担過大 iPhone 上で完結する自己防衛 + 連名告訴接続

2.3　技術アーキテクチャ（30 秒版）

  • フロントエンド：iOS Native (SwiftUI, iOS 17+) — 34 Swift ファイル実装済
  • バックエンド：FastAPI + DuckDB + LanceDB（ベクトル検索）— 公開 GIS API 別途構築中
  • AI：sentence-transformers 多言語埋め込み + 端末上 SLM/LLM (Ollama Llama 3)
  • 暗号：PSI（Private Set Intersection）/ Bloom Filter による被害ハッシュ照合
  • OSS：MIT/Apache 2.0、完全公開（GitHub 上で全コード閲覧可能）

3.4　「自分の事件で動かす」ドッグフーディング実証

開発者本人が当事者の Mapry 事件において、LegalShield のフォレンジック機能を用いて：

  • 61 GB ext4 ファイルシステム鏡像から SHA-256 chain of custody 付きで証拠抽出
  • 1,061 個の証拠ファイルを ハッシュ manifest 化
  • 70 件以上の under-voltage 事件、12 種 PreArm 失敗、176 件 git commit 履歴等を機械的解析
  • 飛行軌跡 GPS 10,410 サンプルを高精度再構築し、「無人機が操縦者頭上を 730 サンプル（2.4 分）通過、最高 154 m から落下した場合 15,180 J ＝ 拳銃弾 30 発相当」を物理計算で立証

これは「実証実験案件」として、トヨタ財団様への申請書類においても研究の実施可能性証明として提示可能です。
