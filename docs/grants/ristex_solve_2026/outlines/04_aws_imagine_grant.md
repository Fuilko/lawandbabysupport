## 3. Project Pitch (English Summary)

**Title**: LegalShield — Cryptographic Evidence Infrastructure for Survivors of Sexual Violence

**Problem**: In Japan, 60% of sexual violence victims never report. The 4% repeat-offender pattern means evidence sits siloed across thousands of unreported cases. Centralized databases re-traumatize victims and create privacy risk.

**Solution**: A three-layer cryptographic infrastructure (built on AWS):
1. **L1 — Self-Sovereign Records**: Client-side encrypted incident logs. Survivor holds the key.
2. **L2 — Federated Cross-Matching**: Multi-Party Computation (MPC) to detect repeat offenders across independent records — without revealing data to any single party.
3. **L3 — Judicial Bridge**: Audit-trail-preserving export to bar associations and law enforcement, only with survivor consent.

**Why AWS**:
- AWS Bedrock for survivor-centric NLP (trauma-informed transcription, multilingual support: JP/EN/ZH/VN)
- AWS Nitro Enclaves for MPC compute
- AWS KMS / CloudHSM for survivor-controlled key management
- AWS Clean Rooms for judicial-grade data exchange
