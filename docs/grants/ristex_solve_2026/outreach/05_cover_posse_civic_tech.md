## ⑤ Code for Japan / 地域 Civic Tech コミュニティ

**件名**：【協力者募集】被害者支援の暗号学的協調ネットワーク（OSS）／5/26-29 東京滞在中

Code for Japan ［担当部署 / ブリゲード］ご担当者様

突然のご連絡失礼いたします。高知大学大学院 修士課程の［氏名］と申します。

私は独立開発者として **被害者が「同じ加害者の別の被害者」と暗号学的に出会える分散型ネットワーク「LegalShield／FEWN」** を、十数年にわたる台湾・日本での現場経験の蓄積を基盤に、近年外部資金ゼロで設計・開発しています（添付 3 ページ紹介書）。

技術スタック：
- **基盤暗号**：OpenMined PSI、SQLCipher、Threshold Cryptography（Shamir 1979）
- **顔エンベディング**：FaceNet/ArcFace の端末ローカル推論
- **データ基盤**：DuckDB ベースの暗数定量化データベース（JUDB）、47 都道府県別ギャップ係数の可視化
- **設計思想**：中央集権プラットフォームではなく、**地域 NPO ごとに暗号鍵を分散保管する自立分散型**

ソースコードは将来的に **完全 OSS 公開** する計画です。MIT または Apache 2.0 を想定。商業化は目的ではなく、暗号モジュールは公益財産として公開し、運用主体（NPO 連合）のみが営業活動を行う構造です。

JST 社会技術研究開発事業（RISTEX）に 6 月 3 日締切で応募準備中ですが、応募の有無にかかわらず、Code for Japan コミュニティの皆様にご相談したいことが 3 つあります：

1. **技術レビュー** — FEWN 設計書の暗号プロトコル・脅威モデル・実装パスを批判的にレビューしていただきたく
2. **OSS コントリビューター募集** — 特に PSI 統合・端末暗号化・GIS 可視化の実装パートに参加していただける方を募集
3. **地域ブリゲード連携** — 高知・大阪・東京等の地域ブリゲードで、本プロジェクトを話題にする勉強会の機会をいただけないか

**5 月 26-29 日東京滞在中** にどなたかと 30 分お話しできれば嬉しいです。Code for Japan Slack や Discord でのチャンネル開設も歓迎します。

私は大企業エンジニアでも研究機関職員でもなく、**「読空気を読まされる職場で疲弊した一人」「制度の隙間から落ちた被害者の一人」として始めた**プロジェクトです。同じ問題意識を持つ技術者の皆さんと、対等に協働したいと思っています。
