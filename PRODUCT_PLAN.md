# 產品開發計畫：法律自救 App + 婦幼醫療 App

> 作者：劉建志 | 建立日期：2026-05-04 | 狀態：規劃中

---

## 背景與動機

在日本偏鄉（高知幡多地區），三大社會痛點：

1. **司法高牆** — 本人訴訟（Honnin Sosho）缺乏系統性支援，律師費高、偏鄉律師極少、複雜案件無人敢接
2. **醫療不均** — 偏鄉夜間小兒急診不足，產婦與新手父母缺乏即時專業判斷
3. **行政碎片化** — 町役所紙本資訊零散、公務員轉介推託（たらい回し），弱勢者無法獲得應有資源

**核心理念：數位賦權（Digital Empowerment）** — 用 AI 消除資訊與資源的不對等。

## 共用底座：三大引擎

### 1. 存證引擎 (Evidence Vault)

所有進入 App 的原始資料（照片、錄音、文字、PDF）必須：

- **SHA256 雜湊** — 產生不可篡改的數位指紋
- **NTP 同步時間戳** — 確保證據時間的法律效力
- **Audit Log** — 記錄每一步操作（誰、何時、做了什麼）
- **加密儲存** — 敏感資料 AES-256 加密

### 2. RAG 檢索引擎 (Knowledge Retrieval)

| 領域 | 資料來源 | 更新頻率 |
|------|---------|---------|
| 法律 | e-Gov、J-Stage、裁判所判例、行政執行命令 | 每日 |
| 醫療 | 日本小兒科學會指引、厚生勞動省通知 | 每週 |
| 行政 | 町役所資源、法テラス、各縣市社福資訊 | 每月 |

**核心規則：AI 輸出的每一句建議，必須強制附帶出處（法條號碼或指南頁碼）**

### 3. SLM/LLM 路由器 (Intelligence Router)

```
使用者輸入
    ↓
隱私分類器 (Privacy Classifier)
    ├── 高隱私 (性侵、個資) → 本地 SLM 處理，不上雲
    ├── 中隱私 (一般法律) → 本地 SLM + 匿名化後可接雲端
    └── 低隱私 (公開資訊查詢) → 直接接雲端 LLM
        ↓
風險分級 (Risk Triage)
    ├── 低: 資料庫檢索 + 自助指南
    ├── 中: 文書草稿 / Triage 建議 (使用者署名)
    └── 高: 自動觸發人工介入 (律師/醫師/119/#8000)
```

## App A: 法律自救 — 功能模組

### A1. 引導式問卷 (Intake)

使用者進入時，透過 5-10 個問題自動分類案型：

- 交通事故 / 傷害
- 性暴力 / 跟蹤
- 不動產糾紛 / 買賣詐欺
- 勞資爭議
- 消費者保護 (PL 法)
- 行政申訴 / 社福申請
- 家事 (離婚、扶養、繼承)
- 跨國貿易 / 智財
- 其他

### A2. 證據採集與保全

- 拍照 → 自動 SHA256 + GPS + timestamp
- 錄音 → 自動轉文字 + 標記關鍵發言
- 文件掃描 → OCR + 語義分類
- 所有證據打包成「數位證據包」，可直接提交法院或 ADR

### A3. 策略模擬 (Multi-Agent Red Team)

- **對方律師 Agent** — 模擬最刁鑽的反駁
- **法官 Agent** — 模擬保守判決傾向
- **顧問 Agent** — 找出使用者論點的邏輯漏洞
- 使用者可反覆修正訴狀直到通過模擬

### A4. 文書自動生成

- 存證信函 (内容証明)
- ADR 申立書
- 準備書面 / 答辯書
- 行政申訴書
- 通報書 (國交省、消費者廳等)

### A5. 轉介與媒合

- AI 判定超出自助範圍 → 自動整理「數位證據包」
- 媒合有心律師 (按專長、地區、意願)
- 連結法テラス等免費資源
- 顯示最近的法律諮詢時間表

## App B: 婦幼醫療 — 功能模組

### B1. 引導式問卷 (Intake)

- 孩子年齡、體重
- 症狀描述 (發燒、嘔吐、皮疹、呼吸異常…)
- 持續時間
- 是否有用藥

### B2. 影像檢傷 (Edge Vision AI)

在 iPhone 上本地執行（離線可用）：

- 皮膚色澤辨識 (發紺、蒼白、黃疸)
- 呼吸頻率估算 (胸腹起伏)
- 囟門狀態 (凹陷 = 脫水)
- 活動力評估

**核心知識來源：妻子（助產士 + 小兒專業）的臨床經驗**

### B3. Triage 輸出

```
綠色 (觀察) → 「目前可在家觀察，建議明早就診」
黃色 (注意) → 「建議 2 小時內就醫，最近的診所是…」
紅色 (緊急) → 「請立即撥打 119 或 #8000」+ 自動撥號
```

### B4. 專家知識蒸餾

- 妻子提供 QA pairs + 判斷邏輯
- 雲端 LLM 擴增為數萬筆訓練資料
- 蒸餾至 1B-3B SLM → CoreML 部署 iPhone
- 定期更新（新生兒指引改版時）

### B5. 急診導航

- GPS 定位最近的夜間急診中心
- 自動撥號 #8000 (小兒救急) / 119
- 產生就醫摘要給醫師（症狀時間線 + AI 判斷）

## 免責架構 (Liability Shield)

### 開發者端

| 防禦層 | 實作方式 |
|--------|---------|
| **定位聲明** | App 是「輔助工具」，非法律代理 / 非醫療診斷 |
| **強制溯源** | 每句 AI 建議必須引用法源或醫學指南 |
| **使用者署名** | 所有法律文書由使用者最終確認署名 |
| **開發審計** | GitHub 記錄所有安全對策的設計決策 |
| **資料不持有** | 敏感資料加密存於使用者裝置，伺服器不留原始資料 |

### 使用者端

| 防禦層 | 實作方式 |
|--------|---------|
| **證據完整性** | SHA256 + timestamp 保全，對手無法質疑造假 |
| **操作紀錄** | Audit Log 證明使用者已盡注意義務 |
| **萬全準備** | 多 Agent 模擬對抗，預先堵住邏輯漏洞 |
| **即時升級** | 高風險自動轉人工，不讓使用者獨自面對危機 |
