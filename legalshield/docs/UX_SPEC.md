# LegalShield UX/UI 仕様書

## 設計原則

> **「バックエンドはプロ級、フロントエンドはおばあちゃんでも使える」**

- **隱蔽性**: スマホを見ているだけで「被害者支援アプリ」とは分からない
- **一鍵性**: 緊急時は 1 タップで行動
- **語音性**: 高齢者・障害者は文字入力不要
- **段階性**: 情報は必要に応じて開示、圧倒しない

## ユーザー層別デザイン

### 層 A: 高齢者版（60歳以上）

**特徴**
- 視力低下、細かい文字が読めない
- スマホ操作に不慣れ
- 詐欺電話・訪問販売の標的
- 一人暮らしで孤立しがち

**UI 仕様**

```

│                                      │
│  ┌─────────────────────────────┐    │
│  │                             │    │
│  │      📞 家族に連絡           │    │  ← 40% of screen
│  │      [お母さんの写真]        │    │
│  │                             │    │
│  └─────────────────────────────┘    │
│                                      │
│  ┌─────────────────────────────┐    │
│  │                             │    │
│  │      🚨 助けて               │    │  ← 40% of screen
│  │      110 / 家族 / 位置       │    │
│  │                             │    │
│  └─────────────────────────────┘    │
│                                      │
│  ⚠️ 午後 2:30 怪しい電話がありました   │
│                                      │
│  [話す]  [カメラ]  [家族一覧]        │

```

**機能**
- **一鍵求救**: 赤ボタン長押し 2 秒 → 110 + 家族 + 位置情報 同時送信
- **詐欺防護**: 電話着信時 AI がリアルタイム分析 → 「詐欺の可能性 85%」警告
- **語音操作**: 「お母さんに電話して」→ 自動発信
- **大文字**: 全テキスト 24px 以上
- **高コントラスト**: 白背景 + 黒文字（#000000）

## 證據収集フロー

```
【被害者視點】

1. 写真を撮る
   → AI: 「傷害写真を検出。タイムスタンプを付与。
          顔が映っています。診断書も取得してください。」

2. 訊息をスクリーンショット
   → AI: 「威脅的な内容を検出。送信元を特定。
          これは刑法 222 條（脅迫罪）に該当する可能性。」

3. 録音をする
   → AI: 「音声をテキスト化。怒鳴り声・威脅を検出。
          一部の都道府県では同意が必要です。
          注意事項を表示。」

4. 醫療受診
   → AI: 「最寄りの指定病院を案内。
          『DV 被害です』と伝えるよう促す。
          診断書の取得を確認。」

5. フォルダ自動生成
   → AI: 「すべての証拠を整理しました。
          証明書を発行。USB と クラウドにバックアップを促す。」
```

## 技術仕様

### フロントエンド
- **フレームワーク**: React Native（iOS/Android 両対応）
- **Web 版**: React + TypeScript + TailwindCSS
- **PWA**: オフライン対応

### バックエンド
- **API**: FastAPI (Python)
- **向量 DB**: LanceDB / FAISS
- **LLM**: Ollama (ローカル) + OpenAI API (フォールバック)
- **ファイル保存**: 暗号化 SQLite + クラウドオプション

### セキュリティ
- **通信**: TLS 1.3
- **保存**: AES-256-GCM
- **認証**: 顔認識 or PIN（パスワード不要）
- **証拠**: SHA-256 ハッシュ + タイムスタンプ + GPS

## アクセシビリティ対応

| 障害類型 | 対応 |
|---------|------|
| 視覚障害 | スクリーンリーダー対応、VoiceOver/TalkBack |
| 聴覚障害 | 振動通知、字幕表示、手話ビデオ連携 |
| 肢体不自由 | スイッチコントロール、音声操作 |
| 認知障害 | シンプル UI、一画面一アクション |
| 高齢者 | 大文字、高コントラスト、音声優先 |
