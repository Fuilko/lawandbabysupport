1. 自我介紹 — LegalShield 是什麼？

LegalShield 是一款以 AI 為核心、專為犯罪被害人與權利侵害被害人設計的免費支援系統。

日本每年超過 200萬人 遭遇犯罪或權利侵害，但其中 70% 因為「不知道找誰談」「證據來不及保存」「害怕被吃案」而選擇獨自承受。LegalShield 的誕生，就是為了打破這道無形的牆。

核心理念
被害人不需要成為法律專家才能保護自己。
AI 協助整理狀況、保全證據、分析法律、模擬策略、媒合專家——
讓每一個被害人，都能在事件發生的「黃金時間」內，做出正確的下一步。

3. 為什麼開發 LegalShield？

開發者自身就是被害者。

2026年2月3日，開發者在台灣進行無人機測量作業時，因為產品設計缺陷導致墜毀。 調查過程中發現製造商隱瞞了 34 項安全缺陷，而行政機關、消費者保護體系對此束手無策。 這段經歷讓開發者深刻體會到：

「當公權力無法保護普通人的時候，
科技應該成為弱者手中的武器。」
三個核心觀察
📉
觀察一：制度缺口
日本有 200萬+ 年度被害人，但法テラス是平日10-16時、NPO 分布不均、 警察吃案頻繁。被害人在「事件發生後的 24-72 小時黃金時間」內， 幾乎得不到結構化支援。
💻
觀察二：技術落差
企業用 Legal Tech（如 Legalscape）已經成熟，但都是給「有錢請律師的企業」用的。 普通人、被害人、弱勢群體，反而被科技拋在後面。
🧠
觀察三：AI 的可能性
2025-2026 年，AI（LLM）已經能夠進行高品質的法律文書分析、 策略模擬、多語言對話。這個能力不應該只屬於大企業， 而應該開放給每一個需要幫助的人。
⚖️
觀察四：防吃案的迫切性
日本警察的「被害屆不受理」（吃案）是長年惡疾。 被害人語無倫次、沒帶證據、不懂法條，結果被警察一句「這只是民事」打發。 如果能事先整理好法律根據，吃案率可以大幅下降。

5.1 如何將資料庫變成 APP？

Step 1: 選擇 APP 框架

框架優點缺點推薦度
Flutter一套程式碼跑 iOS + Android、效能好、UI 靈活Dart 語言需要學習⭐⭐⭐⭐⭐
React NativeJavaScript 生態系、開發者多、函式庫豐富效能略遜、版本碎片化⭐⭐⭐⭐
PWA (Web APP)最快上線、無需審核、網頁即 APP原生功能調用受限⭐⭐⭐⭐（MVP 首選）

Step 2: 端末內 AI 推論（離線可用）

被害人的手機可能在地下室、飛行模式、網路不穩的地方。
使用 ONNX Runtime Mobile 或 TensorFlow Lite 將 AI 模型轉換為端末可執行格式。
5重角色推論完全在端末完成，不傳任何錄音・照片・位置到伺服器。

Step 3: 向量資料庫端末化

LanceDB 支援嵌入式、無伺服器模式。
將 623K 國法向量資料壓縮為 ~500MB 的端末資料庫。
首次下載 APP 時同步，後續增量更新。
法律檢索完全在端末內完成，零網路延遲。

Step 4: 敏感資料端末加密

證據照片・錄音・對話紀錄：AES-256-GCM 端末內加密
金鑰 = 使用者密碼 + 硬體綁定（Secure Enclave / Keystore）
伺服器永遠無法解密。即使伺服器被駭，資料也是廢紙。

6. 如何調用手機原生功能？

LegalShield 需要調用 GPS・相機・麥克風・SMS 等手機功能。以下是各功能的實作方式：

6.1 GPS 位置情報

用途：最鄰近支援機構検索・緊急時位置送信・證據 GPS 標記

平台API / 套件程式碼範例
Flutter `geolocator` 套件 Position pos = await Geolocator.getCurrentPosition();
double lat = pos.latitude;
double lng = pos.longitude;
React Native `@react-native-community/geolocation` Geolocation.getCurrentPosition(pos => {
const {latitude, longitude} = pos.coords;
});
PWA (Web) Geolocation API navigator.geolocation.getCurrentPosition(pos => {
const {latitude, longitude} = pos.coords;
});
⚠️ 隱私保護原則
• GPS 座標不儲存伺服器，僅端末內用於検索最鄰近設施
• 緊急時 SMS 僅發送都道府県・市區町村級別的模糊位置
• 使用者可隨時關閉位置權限，APP 仍可離線使用其他功能
6.2 相機（證據拍照）

用途：受傷照片・事故現場・器物損壊・LINE 截圖存證

平台API / 套件說明
Flutter `image_picker` + `camera` 拍照後自動加上時間戳記 + GPS 座標 + SHA-256 雜湊值的水印
React Native `react-native-image-picker` 同樣自動加上不可竄改的中繼資料（EXIF + 數位簽章）
PWA `<input type="file" accept="image/*">` 受限於瀏覽器沙盒，建議用於 Web MVP，正式版請用原生 APP
6.3 麥克風（錄音・語音輸入）

用途：事件陳述錄音・通話錄音（同意下）・AI 語音對話

平台API / 套件說明
Flutter `flutter_sound` + `speech_to_text` 錄音同時即時語音轉文字，AI 自動關鍵詞抽取
React Native `@react-native-voice/voice` Google Speech-to-Text 連携，日語高精準度
🔒 錄音隱私設計
• 錄音檔不上傳伺服器，端末內 Whisper 模型轉文字後刪除
• 僅文字摘要上傳（「DV・傷害・昨日・夫」等匿名化關鍵詞）
• 通話錄音需雙方同意（APP 內顯示同意確認對話框）
6.4 SMS 簡訊（緊急通知）

用途：10秒倒數後自動送信賴聯絡人・支援機構

平台API / 套件說明
Flutter `flutter_sms` sendSMS(message: msg, recipients: ["080-XXXX-XXXX"]);
React Native `react-native-sms` 同上，調用系統 SMS 發送
📱 無音模式（Silent Mode）
• 緊急 SMS 靜音發送，不發出任何聲音或震動
• 避免加害者發現被害人正在求救
• SMS 內容範本：
「【自動送信】○○（名前）が緊急事態に遭遇しました。
位置：東京都○○区（GPS座標は送信されません）
時間：2026/05/14 14:32
このメッセージは自動生成されたものです。」
