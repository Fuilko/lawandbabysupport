# LegalShield 統合分析レポート — 拡張犯罪分類学 + 加害者側写分析

> **作成日**: 2026年5月14日
> **対象**: LegalShield AI 被害者支援システム
> **範囲**: 17類型の犯罪分類 + 加害者側写データベース + J-STAGE 法律論文分析

---

## 執行摘要

本レポートは、LegalShield の犯罪分類学を従来の5類型から**17類型**に拡張し、各タイプについて **加害者の心理プロファイル・行動パターン・常用狡弁・反論キット** を体系的に整理したものです。

さらに、**J-STAGE（日本学術論文データベース）**から法律・犯罪学関連論文を検索・分析し、学術的知見をシステムに統合する道筋を示します。

## 第2部: 加害者側写分析（Perpetrator Profiling）

### 2.1 分析フレームワーク

```
加害者側写 = {
  心理プロファイル: {
    core_traits,        // 核心的人格特徴
    attachment_style,   // 愛着スタイル
    common_disorders,   // 関連精神障害
    trigger_factors,    // 引き金となる要因
  },
  行動パターン: [...],   // 典型的行動列
  支配戦術: {...},      // 具体的支配手法
  常用狡弁: {...},       // 言い訳パターンと分析
  反論キット: [...],     // 被害者用反論文例
  リスク指標: [...],     // 危険度サイン
  関連法條: [...],       // 該当する法條
}
```

### 2.2 横断的狡弁パターン（全犯罪タイプ共通）

| パターン | 典型例 | 心理メカニズム | 反論の核心 |
|---------|--------|--------------|-----------|
| **被害者責任転嫁** | 「お前が悪い」「お前が先に」 | 自己正当化、認知的不協和の解消 | 被害者に過失があっても、違法行為は別問題 |
| **愛情・友情の装い** | 「愛してるから」「心配だから」 | 支配を善意で隠蔽 | 相手の意思に反する行為は、どんな名目も犯罪 |
| **軽視・矮小化** | 「ただの冗談」「大げさ」 | 被害者の苦痛を認めると自己否定的になる | 苦痛の大きさは被害者が決める |
| **権利の濫用主張** | 「発言の自由」「正義」 | 法律概念の歪曲使用 | すべての権利には責任が伴う |
| **無知の主張** | 「知らなかった」「気づかなかった」 | 故意性の否定 | 常識的・確認義務的に知るべきことは免責にならない |

### 2.3 各タイプの加害者側写サマリー

#### DV（配偶者暴力）
- **核心特徴**: 支配欲、所有欲、ナルシシズム、拒否不耐性
- **愛着スタイル**: 執着型または回避型
- **支配戦術**: ガスライティング、経済虐待、孤立化、脅迫（物損・自殺ちらつかせ）
- **危険サイン**: 別れ話後のエスカレーション、武器購入、ストーカー併発

#### 性暴力
- **核心特徴**: 支配欲、共感能力欠如、権力志向
- **愛着スタイル**: 回避型または混乱型
- **支配戦術**: 否認（同意があった）、矮小化、被害者責任、脅迫（暴露）
- **危険サイン**: 複数被害者、職権濫用、児童接近

#### ストーカー
- **核心特徴**: 執着、拒否不耐性、現実検討能力低下
- **愛着スタイル**: 執着型愛着障害
- **支配戦術**: ラブボミング、監視（GPS・SNS）、孤立化、脅迫
- **危険サイン**: 別れ後エスカレーション、新交際相手への攻撃、家族への接近

#### サイバー犯罪
- **核心特徴**: 匿名性依存、共感能力欠如、優越感、集団心理
- **支配戦術**: ドッグパイリング（集団攻撃）、ドクシング（個人情報暴露）、ガスライティング
- **危険サイン**: 個人情報暴露、就職先への中傷、自殺教唆

## 第3部: J-STAGE 法律論文分析

### 3.1 J-STAGE とは

- **運営**: 科学技術振興機構（JST）
- **規模**: 4,000+ 学術誌、2,400+ 出版社
- **法律関連**: 法哲学年報、法制史研究、犯罪学雑誌等

### 3.2 検索・分析モジュール

| モジュール | ファイル | 機能 |
|-----------|---------|------|
| 検索クローラー | `crawlers/jstage_search.py` | J-STAGE検索、論文メタデータ取得 |
| 分析エンジン | `backend/jstage_analyzer.py` | キーワード抽出、犯罪タイプ分類、トレンド分析 |

```
LegalShield/

│   ├── jstage_analyzer.py         // J-STAGE 論文分析

├── docs/

└── knowledge/
    └── seeds/
        └── jstage_legal_papers.json // J-STAGE 論文データ
```

### 4.2 AI 対話フロー（加害者側写統合版）

```
被害者入力: 「元彼が『心配だから』って毎日家の前に来る」
    ↓
AI 犯罪タイプ判定: ストーカー（確率: 92%）
    ↓
AI 加害者側写参照: {
    心理: 執着型愛着障害、拒否不耐性
    狡弁: 「心配だから」→ 支配の装い
    反論: 「心配を理由にした追跡は法律上ストーカー行為です」
}
    ↓
AI 被害者応答生成: {
    共感: 「それは本当に恐ろしい状況です」
    法的信息: ストーカー規制法・刑法222条
    反論キット: 相手が言うかもしれない言い訳と反論
    行動プラン: 警察相談・証拠保全・保護命令申請
}
```

## 第5部: 戦略的示唆

### 5.1 研究・政策への示唆

| 課題 | J-STAGE 分析からの示唆 | LegalShield の対応 |
|------|----------------------|-------------------|
| サイバー犯罪論文の不足 | 実社会の被害急増に対し、学術研究が追いついていない | AI で被害者支援を先行実装し、データで研究を促進 |
| 被害者支援論文の偏在 | 「被害者の権利」より「犯罪者の量刑」論文が多い | AI 対話で「被害者視点」を補完 |
| 判例分析の活用 | 不同意性交等罪改正後の判例蓄積が進行中 | 最新判例を RAG でリアルタイム検索 |

### 5.2 次期開発優先順位

| 優先度 | 開発項目 | 目的 |
|--------|---------|------|
| P0 | perpetrator_profiler.py の AI 統合 | 対話中に自動で加害者側写・反論を提示 |
| P1 | J-STAGE ベクトル化 | 学術論文を RAG 検索対象に追加 |
| P2 | 狡弁音声認識 | 被害者が加害者の言葉を入力すると、AI が即座に反論生成 |
| P3 | リスクスコアリング自動化 | 被害者の説明から危険度を自動判定 |
