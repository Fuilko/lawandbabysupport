# LegalShield 緊急モード設計仕様書（法的免責・安全設計版）

## 設計原則

> **「AI は準備する。最後のボタンは、被害者が押す。」**

- **User-in-the-Loop**: 自動通報は行わず、被害者が最終決定する
- **Tiered Escalation**: 3 段階の段階的エスカレーション
- **Silent First**: 最初は音を出さない（加害者に気づかれない）
- **Legal Shield**: 明確な免責聲明 + 同意書 + 責任境界

## 10秒カウントダウン機構の詳細

### 問題提起
> ユーザーが操作不能な状態になった場合、誰に自動送信するべきか？
> - 選択肢 A: 家族・友人（身内）
> - 選択肢 B: 公的機関・NPO（専門家）

### 結論：ハイブリッド階層送信（推奨設計）

```
【段階 1: 10秒カウントダウン】
    ↓ キャンセルなし
【段階 2: サイレント SMS 自動送信】
    → 送信先 A: ユーザー設定「緊急連絡先」（最大3名）
    → 送信先 B: 最近隣の民間支援センター（NPO/シェルター）
    → 送信先 C: （オプション）法テラス相談窓口
    ↓ 30秒経過後もキャンセルなし
【段階 3: 警報音 + 画面フルロック】
    → 画面を消しても警報音が止まらない（第三者の注意喚起）
    → ボタン長押しで手動110発信
```

### なぜ「家族」ではなく「緊急連絡先」なのか

| 送信先 | メリット | デメリット | 採用 |
|--------|---------|-----------|------|
| **家族（血族）** | 感情的理解 | DVでは加害者が同居、通報で報復の恐れ | ❌ |
| **友人・信頼者** | 迅速、信頼関係 | 対応能力にばらつき | ✅ 設定可能 |
| **NPO/シェルター** | 専門対応、匿名可 | 夜間・休日の対応限界 | ✅ 自動選択 |
| **警察(110)** | 最強の公權力 | AI誤判定→偽計業務妨害のリスク | ⚠️ 手動のみ |

### 2. 利用規約の核心條項（ToS Core Clauses）

```markdown
## 第 X 條（緊急通報機能に関する免責）

1. LegalShield の緊急通報機能は、
   使用者が自主的に操作する「補助ツール」です。

2. 緊急通報の最終決定権・実行責任は、
   常に使用者本人にあります。

3. 当社は、以下の事由により生じた損害について、
   一切の責任を負いません：
   (a) 通信回線の混雑・途切れ
   (b) 端末の電池切れ・故障
   (c) AI の誤判定
   (d) 第三者（支援機関）の対応遅延・不作為
   (e) 使用者が緊急通報を行わなかったこと

4. 当社は、日本民法第 698 條（緊急事務管理）に基づき、
   悪意又は重大な過失がない限り、
   緊急通報関連の損害賠償責任を免れます。
```

## 各犯罪タイプ別の緊急モード最適化

### DV 被害者向け
```
【緊急ボタン設計】
- ボタン名: 「助けて」
- 色: 赤 → 暗い赤（周囲に光で気づかれない）
- 音: 完全無音
- 画面: 暗転（ブラウザ風に見せかける）
- SMS送信先: シェルター優先（家族は報復の恐れ）
```

### 性暴力被害者向け
```
【緊急ボタン設計】
- ボタン名: 「今すぐ相談」
- 色: 青（落ち着いた色）
- 音: 無音
- 画面: 一般アプリ風（周囲にバレない）
- 初期表示: 「あなたは安全な場所にいますか？」
- SMS送信先: ワンストップ支援センター
```

### 高齢者向け（特殊詐欺）
```
【緊急ボタン設計】
- ボタン名: 「孫の電話」→ 実は支援センターへ
- 色: 孫の写真表示（周囲に見えても自然）
- 音: 通常の着信音
- 画面: 電話アプリ風
- SMS送信先: 自治体高齢者福祉課 + 家族
```

## バックエンド監視とプライバシー境界

### 監視可能な範囲（開発者側）

| データ | 可視性 | 目的 | 保存期間 |
|--------|--------|------|---------|
| 緊急ボタン押下回数 | ✅ 可視 | サービス改善 | 90日 |
| GPS都道府県（市区町村まで） | ✅ 可視 | 最寄り施設検索 | 24時間 |
| 音声・画像・メッセージ内容 | ❌ 不可視 | プライバシー保護 | 端末のみ |
| 個人情報（氏名・住所） | ❌ 不可視 | 匿名設計 | 保存しない |

### プライバシーバイデザイン

```
【端末側処理（オフライン優先）】
音声認識 → 端末内 AI → キーワード抽出のみをサーバー送信
                                    ↓
                          （内容全文は送信しない）
                                    ↓
                          サーバー: 「緊急度: HIGH」だけを受信
                                    ↓
                          サーバー: 最寄り施設情報を返送
```
