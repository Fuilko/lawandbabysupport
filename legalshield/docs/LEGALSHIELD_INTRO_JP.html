1. 自己紹介 — LegalShield とは？

LegalShield は、AI を中核とした犯罪被害者・権利侵害被害者向けの無料支援システムです。

日本では年間 200万人以上 が犯罪や権利侵害に遭いますが、その 70% が「誰に相談すればいいかわからない」「証拠を保存する時間がない」「被害届が不受理になるのが怖い」と、1人で抱え込んでいます。LegalShield は、その無形の壁を壊すために生まれました。

核心理念
被害者が法律の専門家になる必要はありません。
AI が状況を整理し、証拠を保全し、法律を分析し、戦略をシミュレーションし、専家を紹介する——
すべての被害者が、事件発生後の「黄金時間」内に、正しい次の一歩を踏み出せるように。
🛡️
24時間・無料・匿名
本名や住所の登録不要。スマホを開けばすぐに使えます。
🤖
AI 5重ロールシステム
緊急対応、証拠保全、法律分析、戦略シミュレーション、専家紹介をワンストップで。
🔥
被害届不受理防止機能
AI が法的根拠付きレポートと警察交渉スクリプトを生成し、被害届不受理を防ぎます。

⚠️ 起爆剤：6800万円の企業不正被害
2026年2月3日、私が林業測量のために購入したドローンが、製品の設計缺陷により台湾で墜落しました。
調査の過程で、メーカーが34 項の安全缺陷を隠蔽していたことが判明しました——無効化されたバッテリー保護回路、安全限界の5倍を超える磁気センサー、そして製品として出荷された「0.0.0」の開発版ファームウェア。
行政機関・消費者保護体制は無力でした。メーカーは「被害届不受理」のように責任をはねつけ、供給打切りを恫喝しました。
高知・大阪・東京と十数社の法律事務所を探しましたが、手頃な費用で私を助けてくれる専門家は見つかりませんでした。
現在、台湾と日本で同時に訴訟・ADR を進行中（背信・商品損壊）。

この6800万円の惨禍が、すべての起爆剤となりました。
私はドローンのファームウェアを1行ずつ逆アセンブルし、日本の民事法と行政法を1つずつ分析し、数千ページに及ぶ法的証拠データベースを自ら構築しました。
この悪夢がなければ、お金も知識も味方もないまま司法の壁の前に立たされる一般市民の絶望を、私は決して理解できなかったでしょう。

4つの核心観察
📉
観察一：制度の隙間
日本には年間200万+ の被害者がいるが、法テラスは平日10-16時、NPO は地域格差があり、 警察による被害届不受理は頻繁。被害者は「事件発生後 24-72 時間の黄金時間」内に、ほとんど構造化された支援を受けられない。
💻
観察二：テクノロジーの格差
企業向け Legal Tech（例：Legalscape）は成熟しているが、対象は「弁護士を雇える企業」。 一般市民・被害者・弱者こそがテクノロジーから取り残されている。
🧠
観察三：AI の可能性
2025-2026 年、AI（LLM）は高品質な法律文書分析、戦略シミュレーション、多言語対話が可能になった。 この能力は大企業だけのものではなく、すべての助けを必要とする人に開放されるべきだ。
⚖️
観察四：都市と地方の格差
高知県のような地方では、弁護士・NPO・支援機関へのアクセスが極端に限られる。 女性被害者、子育て中の親、高齢者にとって、東京や大阪の資源は遠く、交通費・時間的制約も圧倒的。 デジタルで法の力を地方に届けることが急務だ。

5.1 データベースを APP にするには？

Step 1: APP フレームワークの選択

フレームワークメリットデメリット推奨度
Flutter1つのコードで iOS + Android、高性能、UI 柔軟Dart 言語の学習が必要⭐⭐⭐⭐⭐
React NativeJavaScript エコシステム、開発者が多い、ライブラリ豊富性能やや劣る、バージョン断片化⭐⭐⭐⭐
PWA (Web APP)最速リリース、審査不要、Web 即 APPネイティブ機能制限あり⭐⭐⭐⭐
MVP 最適

Step 2: 端末内 AI 推論（オフライン使用可能）

被害者のスマホは地下室・飛行機モード・電波不安定な場所にいる可能性があります。
ONNX Runtime Mobile または TensorFlow Lite で AI モデルを端末実行形式に変換。
5重ロール推論は完全に端末内で完結し、録音・写真・位置情報は一切サーバーに送信されません。

Step 3: ベクトルデータベースの端末化

LanceDB は組み込み・サーバーレスモードをサポート。
623K 国法ベクトルデータを ~500MB に圧縮した端末データベースに。
初回ダウンロード時に同期、その後は増分更新のみ（週 ~10-50MB）。
法律検索は端末内で完結し、ネットワーク遅延ゼロ。

Step 4: 機密データの端末内暗号化

証拠写真・録音・対話記録：AES-256-GCM 端末内暗号化
鍵 = ユーザーパスワード + ハードウェアバインド（Secure Enclave / Keystore）
サーバーは永遠に復号できません。たとえサーバーが侵害されても、データは無意味です。

6. スマホのネイティブ機能を呼び出すには？

LegalShield は GPS・カメラ・マイク・SMS 等のスマホ機能を利用します。各機能の実装方法は以下の通りです。

6.1 GPS 位置情報

用途：最寄り支援機関検索・緊急時位置送信・証拠 GPS タグ付け

プラットフォームAPI / パッケージコード例
Flutter `geolocator` Position pos = await Geolocator.getCurrentPosition();
double lat = pos.latitude;
double lng = pos.longitude;
React Native `@react-native-community/geolocation` Geolocation.getCurrentPosition(pos => {
const {latitude, longitude} = pos.coords;
});
PWA (Web) Geolocation API navigator.geolocation.getCurrentPosition(pos => {
const {latitude, longitude} = pos.coords;
});
⚠️ プライバシー保護原則
• GPS 座標はサーバーに保存せず、端末内の施設検索のみに使用
• 緊急時 SMS には都道府県・市区町村レベルのぼかし位置のみ送信
• ユーザーはいつでも位置権限を停止可能。権限停止後も他の機能は使用可能
6.2 カメラ（証拠撮影）

用途：傷害写真・事故現場・器物損壊・LINE スクリーンショット保存

プラットフォームAPI / パッケージ説明
Flutter `image_picker` + `camera` 撮影後に自動でタイムスタンプ + GPS 座標 + SHA-256 ハッシュ値の透かしを付加
React Native `react-native-image-picker` 同様に改竄不可能なメタデータ（EXIF + デジタル署名）を自動付加
PWA `<input type="file" accept="image/*">` ブラウザサンドボックスの制限あり。Web MVP 向け。本番はネイティブ APP を推奨
6.3 マイク（録音・音声入力）

用途：事件陳述録音・通話録音（同意の下）・AI 音声対話

プラットフォームAPI / パッケージ説明
Flutter `flutter_sound` + `speech_to_text` 録音と同時にリアルタイム文字起こし、AI が自動でキーワード抽出
React Native `@react-native-voice/voice` Google Speech-to-Text 連携、日本語高精度
🔒 録音プライバシー設計
• 録音ファイルはサーバーにアップロードされず、端末内 Whisper モデルで文字起こし後に音声ファイルを削除
• 文字摘要のみアップロード（「DV・傷害・昨日・夫」等の匿名化キーワード）
• 通話録音は双方の同意が必要（APP 内で同意確認ダイアログを表示）
6.4 SMS ショートメール（緊急通知）

用途：10秒カウントダウン後の信頼連絡先・支援機関への自動送信

プラットフォームAPI / パッケージコード例
Flutter `flutter_sms` sendSMS(message: msg, recipients: ["080-XXXX-XXXX"]);
React Native `react-native-sms` 同上、システム SMS を呼び出して送信
📱 無音モード（Silent Mode）
• 緊急 SMS は静かに送信、一切の音や振動を発しない
• 加害者に被害者が救助を求めていることに気づかれないようにするため
• SMS 本文テンプレート：
「【自動送信】○○（名前）が緊急事態に遭遇しました。
位置：東京都○○区（GPS 座標は送信されません）
時刻：2026/05/14 14:32
このメッセージは自動生成されたものです。」
