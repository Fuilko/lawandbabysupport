# LegalShield 資訊流架構 (Information Flow Architecture)

## 核心理念
> 「讓受害者不會孤身一人。AI 分飾多角，24時間即時協助。」

當一個人遭遇犯罪或權利侵害時，最常見的困境是：
1. **不知道怎麼辦** — 不知道該找誰、該做什麼
2. **不知道證據怎麼收集** — 證據隨時間消失
3. **不知道法律依據** — 不知道自己有什麼權利
4. **不知道哪裡有幫助** — 最近的支援機構在哪裡
5. **害怕獨自面對** — 心理壓力、孤立感

LegalShield 用 AI 扮演多個角色，像一個完整的事務所團隊一樣陪伴受害者。

---

## 五重 AI 角色系統

### 1. Emergency Responder (緊急應變官)
- **觸發**: 用戶打開 APP → 緊急模式
- **功能**: 
  - 即時定位 → 推薦最近的警察署/児童相談所/DV支援センター
  - 一鍵撥打 110 / #7119 / #8898
  - 自動生成位置共享和安全確認訊息
- **輸出**: 緊急聯繫清單 + 距離 + 導航連結

### 2. Evidence Collector (證據保全官)
- **觸發**: 用戶上傳照片/截圖/錄音/文件
- **功能**:
  - 自動上鏈 (timestamp + hash) 生成證據鏈
  - OCR 文字識別 (對話截圖、公文書、契約書)
  - 元數據保全 (EXIF GPS、創建時間、設備資訊)
  - 分類標籤 (脅迫、傷害、金錢詐欺、職場霸凌...)
- **輸出**: 時間軸 + 證據清單 + 保全證明書

### 3. Legal Analyst (法律分析師)
- **觸發**: 用戶描述受害情境
- **功能**:
  - RAG 搜索相關法律條文 (國法 623k chunks)
  - RAG 搜索相似判例 (判例 724k chunks)
  - 匹配都道府県條例 (進行中)
  - 生成「法律依據摘要」→ 告訴用戶哪些法律被違反了
- **輸出**: 相關法條清單 + 判例參考 + 勝訴可能性評估

### 4. Strategy Simulator (戰略模擬師)
- **觸發**: 法律分析完成後
- **功能**:
  - 模擬不同申訴路徑的時間/成本/勝率
    - 路徑 A: 刑事告訴 → 警察
    - 路徑 B: 民事訴訟 → 法院
    - 路徑 C: ADR (調停) → 弁護士会/仲裁機構
    - 路徑 D: 行政申訴 → 消費者庁/労働基準監督署
    - 路徑 E: 民事調停 → 家庭裁判所
  - 計算各路的預期時間、費用、成功率
  - 推薦最佳組合策略
- **輸出**: 路徑比較表 + 推薦策略書

### 5. Referral Navigator (轉介導航員)
- **觸發**: 用戶選擇行動路徑
- **功能**:
  - 基於 GPS 推薦最近/最適合的支援機構
  - 預約輔助 (電話腳本生成)
  - 所需文件清單生成
  - 費用減免條件檢查 (法テラス、生活保護等)
- **輸出**: 機構清單 + 電話腳本 + 準備文件清單

## 數據流

```
[User Input]
    │
    ▼
[情境理解 NLP] ──→ [緊急度判定]
    │                    │
    ▼                    ▼ (高緊急度)
[證據收集] ←────── [緊急 Responder]
    │                    │
    ▼                    ▼
[法律 RAG 檢索] ←── [向量 DB: 國法 623k + 判例 724k]
    │
    ▼
[策略模擬] ←────── [統計 DB: 885 e-Stat + 47 都道府県指數]
    │
    ▼
[轉介導航] ←────── [地理 DB: 支援機構 + 弁護士会 + 施設]
    │
    ▼
[行動計劃書] ────→ [User 執行 + AI 跟進提醒]
```

## 核心技術棧

| 層 | 技術 | 數據源 |
|---|---|---|
| 向量檢索 | SentenceTransformers + FAISS/Annoy | 國法 623k, 判例 724k |
| 結構化查詢 | DuckDB | e-Stat 885 tables, 複合指數 |
| 地理搜索 | 都道府県中心座標 + jageocoder (未來) | 支援機構, 弁護士会 |
| 證據保全 | SHA256 + 時間戳 + IPFS (未來) | 用戶上傳文件 |
| LLM | Ollama (本地) / OpenAI API | RAG 增強回答 |

---

## 隱私・安全設計

- **本地優先**: 證據文件本地加密，不上傳雲端
- **匿名模式**: 用戶可不登入使用大部分功能
- **可擦除**: 一鍵銷毀所有本地數據
- **法律援助特權**: 與弁護士通訊加密保護

## 受害者旅程 (Victim Journey)

```
Day 0  事件發生 ──→ 打開 APP ──→ 緊急模式/證據即拍即存
   │
Day 1  冷靜下來 ──→ 情境描述 ──→ AI 法律分析 (相關法條+判例)
   │
Day 2  考慮行動 ──→ 策略模擬 (5 條路徑比較)
   │
Day 3  決定行動 ──→ 轉介導航 (最近的機構 + 預約腳本)
   │
Day 7  持續跟進 ──→ AI 提醒 (證據補充、期限提醒、心靈支援)
   │
Day 30 進展追蹤 ──→ 更新策略、追加法律依據
```
