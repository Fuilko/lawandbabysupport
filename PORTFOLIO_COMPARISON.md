## 八、戰略優先順序

### 方案 A：先做林業 (穩紮穩打)

```
Q1: 🌲 ForestScanner MVP (已有 pipeline 基礎)
Q2: 🌲 ForestScanner v1.0 + 共用底座抽取
Q3: 👶 PocketMidwife MVP (底座複用 + 妻子 QA 收集)
Q4: ⚖️ LegalShield MVP (底座複用 + 法規知識庫)
```

- ✅ 風險最低、已有 SaaS 和程式碼
- ✅ B2G 標案可帶來穩定現金流
- ❌ 營收啟動最慢

### 方案 B：先做醫療 (最快見效)

```
Q1: 👶 PocketMidwife MVP (最簡單、妻子即 domain expert)
Q2: 👶 v1.0 + 共用底座抽取
Q3: 🌲 ForestScanner MVP
Q4: ⚖️ LegalShield MVP
```

- ✅ 開發最快 (~2 個月 MVP)、社會影響力最大
- ✅ App Store 上架後口碑傳播快
- ❌ 醫療法規風險最高

### 方案 C：先做法律 (最大市場)

```
Q1: ⚖️ LegalShield MVP (Web 版)
Q2: ⚖️ v1.0 + 共用底座抽取
Q3: 👶 PocketMidwife MVP
Q4: 🌲 ForestScanner MVP
```

- ✅ 市場最大 (¥6,000 億)、個人痛點最深
- ✅ 自己的案件就是最好的 POC
- ❌ 開發最複雜、耗時最長

### 推薦：方案 A (林業優先)

| 理由 | 說明 |
|------|------|
| **1. 已有基礎** | SaaS 平台、panorama_split、ARCHITECTURE.md 都就緒 |
| **2. 風險最低** | 量測工具無法規風險，不涉及人命或法律責任 |
| **3. 底座先行** | 開發林業 App 的過程中建立共用底座 (Edge AI、FastAPI、Docker) |
| **4. 現金流** | B2G 標案 > B2C 訂閱（初期更穩定） |
| **5. 驗證能力** | 成功上線一個產品 → 證明團隊能力 → 更容易拿到醫療/法律的合作 |
