**「偽計」具体例**:
1. D-31 故意未完成出荷
2. D-32 ZED-F9P RTK 機能偽装

## 9. システム改進（同時並行で実施）

### 9.1 evidence_gate priority 拡張（v1.1）

v1 で「全ファイル必読」設計したため flylog 1,636 件全件が pending になり利用者から「raw data は forensic agent 専用」と指摘。

**改善方針**:
```python
class FileRole(Enum):
    REPORT = "report"        # 分析報告 .md, .pdf → LegalShield agent 必読
    EVIDENCE_DOC = "evidence_doc"  # 契約、mail、内容証明 → 必読
    META = "meta"            # PORTFOLIO, README, manifest → 優先読
    RAW_DATA = "raw_data"    # ext4 image, raw log, raw GPS → forensic agent 専用、本 agent 読不要
    VISUALIZATION = "visualization"  # 3D 画像、HTML map → 1 件サンプル読了で OK
    SCRIPT = "script"        # 分析 .py → 読不要（出力 .md を読む）
```

- coverage 計算は REPORT + EVIDENCE_DOC + META のみ
- raw_data / script は自動 mark_read（agent 不要）
- visualization は 1 件読むと残全件自動 mark_read

### 9.2 ワークフロー標準化（AGENTS.md §2.5 拡張案）

新案件受任時の標準手順:
1. PORTFOLIO.md を読み案件の真の所在地特定
2. 案件 dir 全部（複数 source 可）を `index_evidence_folder()` で manifest 作成
3. priority 自動分類
4. REPORT + EVIDENCE_DOC + META のみ read → mark_read
5. `assert_ready_for_analysis(min_coverage=0.90)` パス確認
6. 既存 ACTION_PLAN.md があれば**それを骨格に**して MASTER 作成（重複しない）
7. 分析報告先頭に coverage_banner + 旧版破棄宣言

## 10. 結論と戦略の核心

### 10.1 mapry 案の本質

これは「単純な製品不良訴訟」ではなく、**「悪意ある詐欺 + 危険製品販売 + 国籍差別 + 業務妨害」の複合的不法行為**。

主要根拠:
- D-31 開発者コメント「**仕方ないのでこの文言で行く**」← 故意の文書化
- 三層同時無効化（FCU + ソフト + 通信）= 偶然不可能
- 自動テスト 0 + 5 週間開発 + 「狂乱 reboot loop」= 未完成認識
- mapry 自社員 宮田氏が墜落時操縦中（「劉氏の計測業務」主張は虚偽）
- 「断貨」脅迫の明文化、第三者 CC 公然性確認済の国籍差別言辞
- 自動 `git reset --hard` 証拠隠滅メカニズム

### 10.2 山口個人責任の必達

利用者の明示要件「**斷尾在法人陪葬で済まさない**」を満たすため:
1. **会社法 429 条**を訴状の核心条文として配置
2. 訴状で **mapry 法人 + 山口圭司 共同被告**
3. 刑事告訴は **山口個人を被告人**として進行
4. mapry 破産時の備えとして法人格否認の法理 + 個人資産差押え準備

### 10.3 求償権譲渡契約の戦略的価値

利用者既に契約書草案を準備済。**ADR 不成立を契機に施行**:
- 訴訟適格集約（劉氏単独原告化）
- mapry 律師「利益相反」主張への完全反論
- 管轄問題解決（高知地裁専属合意）
- 弁護士法 72 条非弁活動回避設計済
